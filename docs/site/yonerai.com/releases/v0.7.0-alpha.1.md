## Title

YonerAI v0.7.0-alpha.1 Official Bridge Foundation

## Summary

v0.7.0-alpha.1 is a prerelease bridge foundation after the v0.6.0 stable CLI
Local Runtime. It adds proposal-only self-evolution, bridge visibility in the
interactive CLI, and site/release/press content contracts for future official
surfaces.

It does not enable production Official Managed Cloud, production Oracle,
production Google login, OpenAI shared traffic, live Discord, or a production
network installer.

## Primary command

```powershell
yonerai
```

## Bridge commands

```powershell
yonerai evolve status --pretty --lang ja
yonerai evolve simulate --fixture tests/fixtures/self_evolution/queue_signals.json --pretty --lang ja
yonerai evolve proposals list --fixture tests/fixtures/self_evolution/queue_signals.json --json
yonerai evolve proposals show proposal-tui.slash_help --fixture tests/fixtures/self_evolution/queue_signals.json --pretty
yonerai auth status --pretty --lang ja
yonerai auth google login --dry-run --pretty --lang ja
yonerai privacy status --pretty --lang ja
```

Inside the interactive CLI:

```text
/自己進化
/evolve
```

## Manifest commands

```powershell
# Release ZIP flow: download manifest.v0.7.0-alpha.1.json as a separate asset first.
$manifest = ".\manifest.v0.7.0-alpha.1.json"
yonerai manifest verify $manifest --pretty
yonerai install plan --manifest $manifest --pretty
yonerai update check --manifest $manifest --pretty
yonerai update plan --manifest $manifest --pretty
```

These commands read local files and print verification or dry-run plans. They
do not download, install, mutate PATH, execute remote scripts, request admin,
write services, or connect to a production control plane.

In a checkout, use `$manifest = "releases\manifest.v0.7.0-alpha.1.json"`
instead.
