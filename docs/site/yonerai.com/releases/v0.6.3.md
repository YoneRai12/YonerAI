# YonerAI CLI Local Runtime v0.6.3

This is site-side content for the technical release page
`https://yonerai.com/releases/v0.6.3`.

GitHub Release remains the distribution source of truth:
`https://github.com/YoneRai12/YonerAI/releases/tag/v0.6.3`.

## Install

Quick install:

```powershell
irm https://install.yonerai.com | iex
```

Verified install / GitHub fallback:

```powershell
$ErrorActionPreference = "Stop"
$url = "https://raw.githubusercontent.com/YoneRai12/YonerAI/62ca47c792f7eae693f9346a8cc34fadc17b8c31/install.ps1"
$expected = "e2990bd0cbc35da35388f7338246ca6eaba557f4990606a25bd127c64bc1ba03"
$tmp = Join-Path ([System.IO.Path]::GetTempPath()) ("yonerai-bootstrap-" + [guid]::NewGuid().ToString("N"))
New-Item -ItemType Directory -Path $tmp | Out-Null
try {
  $script = Join-Path $tmp "install.ps1"
  irm $url -OutFile $script
  $actual = (Get-FileHash -LiteralPath $script -Algorithm SHA256).Hash.ToLowerInvariant()
  if ($actual -ne $expected) { throw "install.ps1 hash mismatch" }
  $scriptText = Get-Content -LiteralPath $script -Raw
  if ($scriptText -notmatch "Installer skeleton" -or $scriptText -notmatch "install.ps1 is still plan-only") {
    throw "install.ps1 is not the expected plan-only bootstrap. Refusing to launch."
  }
  & powershell -NoProfile -ExecutionPolicy Bypass -File $script -Execute -Launch
} finally {
  if (Test-Path -LiteralPath $tmp) { Remove-Item -LiteralPath $tmp -Recurse -Force }
}
```

The fallback uses the same pinned digest as the Cloudflare wrapper. Do not pipe
`releases/latest/download/install.ps1` directly into `Invoke-Expression`, and do
not treat a mutable release sidecar hash as the bootstrap authenticity root.

Manifest checks:

```powershell
$manifest = ".\manifest.v0.6.3.json"
yonerai manifest verify $manifest --pretty
yonerai install plan --manifest $manifest --pretty
yonerai update check --manifest $manifest --pretty
yonerai update plan --manifest $manifest --pretty
```

## What changed

- `yonerai.com/install` exposes both Quick install and Verified install.
- The verified install command checks a pinned SHA256 digest embedded outside
  mutable release assets before executing the plan-only bootstrap script.
- `install.ps1` verifies manifest, channel, versioned artifact name, and release
  ZIP SHA256 before install-like steps.
- `yonerai update check --pretty`, `yonerai update plan --pretty`, and TUI
  `/更新` show channel, artifact, trust/signature state, and non-actions.

## Boundary

- `yonerai.com` is a command page only.
- `yonerai.com` must not serve `install.ps1`, SHA256 sidecars,
  `manifest.v0.6.3.json`, or `YonerAI-0.6.3.zip`.
- GitHub Release assets are the distribution source.
- No production signing keys or production trust store are included.
- No production Oracle, Official Managed Cloud runtime, live Discord, or
  production Google login is included.
