# YonerAI CLI Local Runtime v0.7.0

Public content source for `https://yonerai.com/releases/v0.7.0`.
GitHub Release remains the distribution source:
`https://github.com/YoneRai12/YonerAI/releases/tag/v0.7.0`.

## Quick install

```powershell
irm https://install.yonerai.com | iex
```

The short command downloads a Cloudflare static wrapper. The wrapper downloads
`install.ps1` and `install.ps1.sha256` from GitHub Release assets, verifies the
script hash, then lets the GitHub Release bootstrap verify the manifest and ZIP
hash before install-like actions.

GitHub Release fallback:

```powershell
iex "& { $(irm https://github.com/YoneRai12/YonerAI/releases/latest/download/install.ps1) } -Execute -Launch"
```

## Verified install

Use this when you want to verify the bootstrap script before execution. It
downloads `install.ps1` and `install.ps1.sha256` from GitHub Releases, checks
the SHA256 sidecar, and fails closed before execution if the sidecar is missing,
malformed, or mismatched.

## What changed

- Memory UX is now part of the stable CLI Local Runtime lane.
- `yonerai memory status/add/list/forget/sync preview` is available.
- Local memory remains local by default.
- Local-to-cloud sync remains disabled unless a future official lane explicitly
  approves it.
- Agent Console plan/review/subagent preview surfaces are available in the
  interactive CLI.
- v0.6.5 repair-safe installer behavior is preserved.

## Commands

```powershell
yonerai
yonerai chat
yonerai memory status --pretty --lang ja
yonerai memory add "local note" --scope local --pretty --lang ja
yonerai memory list --scope local --pretty --lang ja
yonerai memory sync preview --direction local-to-cloud --pretty --lang ja
yonerai ask "hello" --auto --pretty --lang ja
yonerai update check --manifest manifest.v0.7.0.json --pretty
yonerai update plan --manifest manifest.v0.7.0.json --pretty
```

Release ZIP flow:

```powershell
$manifest = ".\manifest.v0.7.0.json"
yonerai manifest verify $manifest --pretty
yonerai install plan --manifest $manifest --pretty
yonerai update check --manifest $manifest --pretty
yonerai update plan --manifest $manifest --pretty
```

## Boundaries

This is stable for the CLI Local Runtime slice only. It is not full YonerAI
cloud production. It does not include production Oracle, live Discord,
production Google login, production signing/trust, npm/winget, or auto-update
apply. No production signing keys or production signature trust store are
included. GitHub Release assets remain the source for installable bytes.
