# YonerAI v0.11.0-alpha.1

Account Sync and Oracle API Foundation.

This page is site-side content for yonerai.com. GitHub Release remains the
distribution source of truth.

## Run locally

```powershell
yonerai
yonerai auth status --pretty --lang ja
yonerai sync status --pretty --lang ja
yonerai sync preview --direction cloud-to-local --json
yonerai sync approve --dry-run --direction local-to-cloud --json
yonerai privacy status --pretty --lang ja
yonerai sync api-contract --json
yonerai sync rate-limit --json
yonerai manifest verify releases/manifest.v0.11.0-alpha.1.json --pretty
yonerai install plan --manifest releases/manifest.v0.11.0-alpha.1.json --pretty
yonerai update check --manifest releases/manifest.v0.11.0-alpha.1.json --pretty
```

## What this release adds

- Public account identity contract.
- Google auth dry-run session contract.
- Cloud/local sync envelope, decision, and audit contract.
- `yonerai sync` CLI commands.
- `/同期` in the Japanese Mission Control CLI.
- Official API contract fixture for `/v1/account/me`, `/v1/conversations`,
  `/v1/sync/preview`, `/v1/sync/approve`, `/v1/oracle/runs`,
  `/v1/oracle/runs/{id}`, `/v1/rate-limit`, and `/v1/status`.
- Rate-limit policy shape for user/device/provider quota and local fallback.
- Sanitized private YonerAIOracle self-evolution handoff.

## Boundaries

- No production Google login.
- No production Oracle.
- No Official Managed Cloud runtime in the public repo.
- No automatic local-to-cloud sync.
- No OpenAI shared traffic.
- No live Discord.
- No production installer/npm/winget.
- No private file, local memory, local node payload, provider key, or raw prompt
  upload.
