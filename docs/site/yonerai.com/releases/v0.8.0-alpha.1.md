## Title

YonerAI v0.8.0-alpha.1 Official Install/Auth Boundary

## Summary

v0.8.0-alpha.1 is intended to make the public install/auth/privacy/evolution
boundary easier to understand from the CLI and installer pages. It keeps
installation plan-first, Google OAuth dry-run-only, OpenAI shared traffic off,
and self-evolution proposal-only.

It does not enable production Official Managed Cloud, production Oracle,
production Google login, OpenAI shared traffic, live Discord, production
signing/trust, npm/winget distribution, or a production network installer.

## Commands

yonerai

## Installer Boundary

`install.ps1` is plan-only. It may read a local manifest and display artifact,
SHA256, signature, and trust status. It does not download, install, mutate
PATH, execute remote code, request admin, edit registry, or install services.

## Auth and Privacy Boundary

Google auth is a dry-run contract with loopback redirect, PKCE, state, and
minimal scopes. No browser launch, token exchange, token printing, account
linking, or plaintext refresh token storage is performed.

OpenAI shared traffic remains disabled by default. Private prompts, workspace
files, local node payloads, local memory, and secrets are excluded.

## Self-Evolution Boundary

Public self-evolution remains synthetic/proposal-only. It can display proposal
queue state and generate owner-reviewable proposal records from public-safe
fixtures. It cannot mutate code, GitHub, releases, deployments, production
configuration, or production Oracle.

## Not Included

- Full YonerAI cloud production readiness.
- Production Oracle/control-plane runtime.
- Official Managed Cloud runtime in this public repo.
- Production Google login or token storage.
- OpenAI shared traffic runtime.
- Live Discord restoration.
- Production network installer.
- PATH mutation by default.
- Production signing keys or production trust stores.
- npm or winget distribution.
