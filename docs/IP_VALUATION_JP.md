更新日: 2026-03-07

## 1. 結論（エグゼクティブサマリー）

本リポジトリのIP価値は、**「コード資産 + アーキテクチャ設計 + 運用知見 + 拡張基盤」**の複合として評価するのが妥当です。

- **再調達原価（外注で同等品を再構築）ベースのレンジ**: **約 3,000万円〜1.5億円**
- 現状コード・機能群を中心にした「現実的な再現コスト（Current 相当）」: **約 5,000万円〜9,000万円**
- 商用SaaS水準まで含めた「運用・SRE・セキュリティ拡張込み」: **1億円超**も十分あり得る

> 補足: これは企業価値（株式価値）ではなく、技術資産の**再取得コスト / 代替コスト**に主眼を置く評価です。

## 3. 価値算定の方法

本レポートでは、公開情報のみで妥当性を担保しやすい **コストアプローチ（再調達原価法）**を主採用。

- 理由1: 売上・利益・継続率などの事業KPIが未提示
- 理由2: 比較可能な取引事例（完全同等）が希少
- 理由3: 実装規模・技術難度・運用要件から再現コストを推定しやすい

補助的に、IP価値の上振れ要因として以下を加味:

- 設計ドキュメント群の整備
- テスト群
- 拡張機構（skills/MCP/relay/core分離）

## 4. 外注した場合の費用感（日本市場の一般的レンジ）

以下は、受託開発（AI/バックエンド/フロント/DevOps混成）での概算です。

- QA/自動テスト: 70〜120万円 / 人月
- PM: 100〜180万円 / 人月
- DevOps/SRE: 100〜180万円 / 人月

1) **MVP再現（最小）**
- 期間: 4〜6か月
- 体制: 3〜5名
- 総工数: 18〜30人月
- 想定費用: **約 2,500万〜5,000万円**

2) **Current相当（現repo機能をかなり再現）**
- 期間: 7〜12か月
- 体制: 5〜8名

- 想定費用: **約 5,000万〜9,000万円**

## 5. 再現コスト（Reproduction Cost）の内訳

Current相当（5,000万〜9,000万円）を、機能ブロックに分解すると次のイメージです。

- コア対話/ルーティング/履歴管理: 1,000〜2,000万円
- Tool実行基盤 + 承認/監査 + ポリシー: 800〜1,500万円
- Discord統合（音声/イベント/運用導線）: 600〜1,200万円
- Web API + 管理面 + 認証まわり: 700〜1,400万円
- UI（Dashboard + Web Chat）: 600〜1,200万円
- テスト/品質保証/CI整備: 500〜1,000万円

合計: **約 4,500万〜9,000万円**（バッファ込みで 5,000万〜9,000万円）

## 6. このシステムが「IP」になっている根拠

単なる行数ではなく、以下の**再利用可能な設計知**が蓄積されている点がIP価値の核です。

1. **分割統治の設計**
   - Bot / Admin API / Core / UI / Relay の責務分離
2. **拡張点の標準化**
   - Skills, Tools, MCP 連携
3. **安全性設計**
   - 承認・監査・ポリシー・アクセス制御の思想
4. **運用可能性**
   - テスト群とドキュメント群の存在
5. **ハイブリッド志向**
   - ローカルファーストからクラウド拡張までの移行設計

これらは、他社が短期間で模倣しづらい「暗黙知の形式知化」であり、IPとして評価できます。

## 7. 価格レンジを上げる要因 / 下げる要因

### 上振れ要因
- 実運用データ（DAU、継続率、課金、解約率）が強い
- セキュリティ監査済み・SLA運用済み
- 独自データセットや高品質プロンプト資産が整理済み
- 導入先のユースケース適合度が高い

### 下振れ要因
- OSS依存比率が高く差別化が弱い部分
- 特定個人の暗黙知依存（属人性）
- 環境依存（Windows固有、ローカル前提）で横展開が難しい場合
- 収益化導線が未整備

- 直近12か月の開発実績（実人月、工数ログ）

これらが揃えば、コスト法に加えてDCFやロイヤルティ免除法も適用可能になり、評価精度は大きく改善します。

## 10. 投資家向け評価書フォーマット（DCF/ロイヤルティ免除法版）

以下は、投資家説明でそのまま使える「章立てテンプレート」です。

### 10.1 カバーページ

- 件名: YonerAI 技術資産評価書（ドラフト）
- 目的: 資金調達/提携交渉における技術資産価値の説明
- 評価基準日: YYYY-MM-DD
- 対象: 公開repo + 付随ドキュメント + 運用知見（公開範囲）

### 10.2 エグゼクティブサマリー

- コスト法（再調達原価）: 5,000万〜9,000万円（中心レンジ）
- 収益法（DCF）: 前提シナリオ別で算出（後述）
- 収益法（ロイヤルティ免除）: 売上×ロイヤルティ率の現在価値
- 最終レンジ: 各手法の重み付け平均 + リスク調整

### 10.3 DCF（Discounted Cash Flow）

#### 10.3.1 算式

- 企業価値（技術資産由来部分）

`PV = Σ[ FCF_t / (1 + r)^t ] + TV / (1 + r)^n`

`TV = FCF_(n+1) / (r - g)`

#### 10.3.2 変数定義

- `FCF_t`: t年目のフリーキャッシュフロー（技術資産に帰属する分）
- `r`: 割引率（WACCまたはハードルレート）
- `g`: 永続成長率（通常 0〜2% 程度を保守的に採用）

#### 10.3.3 実務の作り方（YonerAI向け）

1. 売上を「ライセンス/受託/保守/利用課金」に分解
2. 技術資産寄与率（例: 40〜70%）を設定してFCFを按分
3. 5年予測 + ターミナルバリューを算出
4. ベース/アップサイド/ダウンサイドの3ケースを作成

#### 10.3.4 参考レンジ（説明用の例）

前提例（あくまで試算テンプレート）:
- 1年目売上: 4,000万円
- 売上成長率: 年 25%（5年間）
- EBITDAマージン: 18%
- 税・再投資考慮後のFCF率: 10%
- 割引率 `r`: 18%
- 永続成長率 `g`: 1%

この場合、技術資産帰属分のDCF現在価値は概ね **6,000万〜1.2億円**帯に入りうる。

### 10.4 ロイヤルティ免除法（Relief-from-Royalty）

#### 10.4.1 算式

`IP Value = Σ[ Revenue_t × Royalty Rate × (1 - Tax Rate) / (1 + r)^t ]`

#### 10.4.2 変数定義

- `Revenue_t`: IPが組み込まれる売上
- `Royalty Rate`: 同種ソフトウェアの仮想ライセンス率
- `Tax Rate`: 実効税率
- `r`: IPリスクを反映した割引率

#### 10.4.3 ロイヤルティ率の目安

- 低位: 3%
- 中位: 5%
- 高位: 8%

※ 汎用OSS寄りなら低位、独自アルゴリズム・運用ノウハウが強ければ中〜高位を採用。

#### 10.4.4 参考レンジ（説明用の例）

前提例:
- 対象売上: 初年度4,000万円、年25%成長、5年
- ロイヤルティ率: 3% / 5% / 8%
- 税率: 30%
- 割引率: 18%

現在価値の目安:
- 3%ケース: **約1,000万〜1,500万円**
- 5%ケース: **約1,700万〜2,500万円**
- 8%ケース: **約2,700万〜4,000万円**

### 10.5 手法統合（投資家説明用）

単一手法だとブレるため、次のような重み付けが実務的です。

- コスト法: 40%
- DCF: 40%
- ロイヤルティ免除法: 20%

これに実行リスク調整（-10%〜+15%）を掛け、最終レンジを提示します。

### 10.6 投資家向け提出物チェックリスト

- 評価前提表（単価・成長率・割引率・税率）
- 感応度分析（`r`、成長率、ロイヤルティ率）
- KPI証憑（利用実績、継続率、商談化率）
- ライセンス整合性資料（後述の矛盾解消案）

## 12. まとめ

- 本repoは、**PoCを超えた統合型AI基盤**として十分な規模と設計資産を持つ。
- 企業外注で同等品を再構築する場合、**最低でも数千万円、実務的には5,000万〜9,000万円**が妥当帯。

- 投資家向けには、**コスト法 + DCF + ロイヤルティ免除法**の3本立てで説明し、前提と感応度を開示するのが有効。
- ライセンスは、まずREADME等の表記矛盾を解消し、必要ならデュアルライセンス化で商用導線を設計する。
