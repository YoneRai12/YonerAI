# YonerAI Staging Google Login E2E Dogfood - 2026-06

Status: public-safe dogfood evidence.

Scope: public CLI against `https://api-staging.yonerai.com`.

This file intentionally omits request IDs, raw email addresses, session hashes,
Google tokens, auth codes, refresh tokens, local paths, and private backend
inventory.

## Command

```powershell
$env:YONERAI_STAGING_AUTH_ORIGIN = "https://api-staging.yonerai.com"
$env:YONERAI_OFFICIAL_API_STAGING_ORIGIN = "https://api-staging.yonerai.com"
yonerai login
```

The dogfood run used the equivalent expert/CI flags against a temporary config
path so the owner profile was not modified:

```powershell
yonerai login --bridge --open-browser --wait-linked --max-wait-seconds 35 --json
```

## Follow-up From Dogfood

- `yonerai rate-limit` initially failed closed because staging returned
  operational metadata in the rate-limit body.
- The CLI now extracts only the public rate-limit/control-spine fields and drops
  known private operational metadata before rendering.
- Unknown token-like fields still fail closed.
