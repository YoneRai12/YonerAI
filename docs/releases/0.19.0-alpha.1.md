## Summary

YonerAI v0.19.0-alpha.1 connects the public CLI staging login path to cloud
conversation sync preview. After staging Google login, the CLI can use an
opaque YonerAI staging session claim to read metadata-only cloud conversation
references and preview cloud-to-local sync decisions.

This is staging-only. It is not production Google login, not production
Official Managed Cloud, not production Oracle/cloud runtime, and not automatic
local-to-cloud upload.

## What Users Can Run

Set the staging origin:

```powershell
$env:YONERAI_STAGING_AUTH_ORIGIN = "https://api-staging.yonerai.com"
```

Run staging login and wait until the CLI bridge links:

```powershell
yonerai auth google login --staging --bridge --open-browser --wait-linked --pretty --lang ja
```

Check the local staging session claim:

```powershell
yonerai auth session status --pretty --lang ja
yonerai auth status --pretty --lang ja
```

Preview cloud conversation sync:

```powershell
yonerai sync status --pretty --lang ja
yonerai sync conversations --pretty --lang ja
yonerai sync conversation show <cloud_conversation_id> --pretty --lang ja
yonerai sync preview --direction cloud-to-local --audit-reason "user_selected_cloud_preview" --pretty --lang ja
yonerai sync preview --direction local-to-cloud --audit-reason "manual_preview_only" --pretty --lang ja
```

Open the interactive screens:

```text
/認証
/同期
/クラウド
```

Clear the local staging session claim:

```powershell
yonerai auth logout --staging --pretty --lang ja
```

## E2E Evidence

Public-safe evidence file:

- `docs/evidence/STAGING_CLOUD_SYNC_PREVIEW_E2E.md`

Verified staging behavior:

- `GET https://api-staging.yonerai.com/v1/status` returned `200`.
- `GET https://api-staging.yonerai.com/v1/rate-limit` returned `200`.
- staging bridge login reached linked state.
- `auth session status` reported linked state with token printing disabled.
- `sync status` reported `cloud_to_local=preview_available`.
- `sync conversations` returned metadata-only cloud conversation refs.
- `sync conversation show` returned a redacted metadata summary only.
- `sync preview --direction cloud-to-local` returned an allowed preview decision.
- `sync preview --direction local-to-cloud` stayed approval-required and did not call the backend.

No account email, request id, auth code, Google token, refresh token, opaque
session value, client secret, provider key, private path, or private runtime
inventory is included in the public evidence.

## Security and Privacy Boundary

- staging origin remains allowlisted
- non-HTTPS public origins are rejected
- credentialed URLs and private IP origins are rejected
- redirects are rejected before following unexpected locations
- public CLI stores only a YonerAI staging session claim, not Google tokens
- Windows persistent session storage uses DPAPI
- pretty/JSON output prints session availability and hash only, not the value
- cloud conversation output is metadata-only
- raw cloud conversation body is not printed
- local private memory, local file content, and local node content are not uploaded
- local-to-cloud stays disabled by default and requires explicit approval
- OpenAI shared traffic remains off
- production Google login remains unavailable

## Known Limitations

- This is an alpha prerelease line, not the stable CLI Local Runtime line.
- Production Google login/passkey/account runtime is not included.
- Production AWS backend is not included.
- Production Oracle/cloud runtime is not included.
- Cloud-to-local is preview only in the public CLI.
- Local-to-cloud upload is not performed.
- OpenAI shared traffic is not enabled.
- Local private memory is not uploaded automatically.
- Production cloud memory sync is not included.
- Production installer signing/trust store is not included.
- npm, winget, and auto-update apply are not included.
- Arbitrary shell/file/tool execution is not enabled.
- `src/cogs/ora.py` remains a legacy boundary and is not solved by this release.
- `reference_clawdbot` is not touched.

| Change | User-visible impact | Boundary |

| Cloud conversation list/show | Users can inspect staging cloud conversation metadata from the CLI. | Metadata-only; raw cloud conversation body is not printed. |
