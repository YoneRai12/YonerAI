# 2026.05.27 — YonerAI CLI Local Runtime v0.6.0

Release date: 2026-05-27
Tag: `v0.6.0`
Status: stable release for the local CLI runtime slice
Previous release: `v0.6.0-alpha.1`

## Summary

YonerAI CLI Local Runtime v0.6.0 promotes the v0.6 terminal experience from
alpha to a stable local CLI runtime slice. It keeps the same zero-trust
boundaries as the alpha while adding post-alpha hardening: CI Quality Wall
coverage, Google OAuth dry-run visibility, OpenAI shared-traffic policy
visibility, safer update/install status, stronger ledger/path redaction, and a
more complete Japanese-first interactive screen.

This is stable for the local CLI runtime only. It is not full YonerAI cloud
production and it is not a production network installer.

## What changed since v0.6.0-alpha.1

- Added the CI Quality Wall and verified Linux, Windows, and macOS local CLI
  checks for core, CLI smoke, TUI smoke, provider boundary, hybrid zero-trust,
  installer manifest, security-static, and release-gate lanes.
- Added Google OAuth dry-run and privacy/shared-traffic status surfaces while
  keeping production OAuth and OpenAI shared traffic disabled.
- Hardened local run ledger file permissions and path redaction.
- Hardened update-check manifest path handling and PowerShell shell detection.
- Ensured network-off interactive mode suppresses live execution.
- Polished the v0.6 TUI home/settings/provider/local-LLM/auth/privacy/update
  screens and preserved Japanese command suggestions while keeping English
  aliases compatible.

## Provider and Local LLM

- Mock provider is ready by default.
- Local LLM setup remains loopback-only: `localhost`, `127.0.0.1`, or `::1`.
- OpenAI-compatible, Anthropic, and Gemini remain opt-in readiness surfaces
  unless explicitly configured with live/provider-specific opt-in.
- Provider keys are not printed, stored in CLI config, or written to run
  history.

## Auth and Privacy

- Google OAuth is contract/dry-run only.
- OAuth state and PKCE requirements are visible, but no live OAuth request,
  browser launch, token exchange, token printing, or refresh-token storage is
  performed.
- OpenAI shared traffic is off by default and not runtime-enabled in this
  release.
- Private prompts, workspace-local file content, local memory records, local
  node payloads, provider keys, and secrets are excluded from shared-traffic
  policy.

## Update and Installer Safety

- `yonerai update check` and `/更新` are local-manifest checks by default.
- `yonerai install plan` and `yonerai update plan` remain dry-run planners.
- The generated ZIP does not include versioned manifest files; the manifest is
  published as a separate GitHub Release asset so it can record the ZIP hash.
- `install-local.ps1` remains explicit local bootstrap, not a production
  network installer.
- `install.ps1` remains a dry-run skeleton for a future installer lane.
- Production signing keys, production trust stores, npm, winget, and remote
  execute install flows are not included.

## Security and Boundary

- No production Oracle/cloud runtime.
- No live Discord or Discord token use.
- No deploy/public tunnel.
- No arbitrary shell/file/tool execution.
- No provider live call by default.
- No provider key output or config/ledger storage.
- Local LLM remains loopback-only.
- Official Managed Cloud remains external/contract-only.
- `src/cogs/ora.py` is not solved by this release.
- `reference_clawdbot` is not touched.

## Known limitations

- Stable applies only to the YonerAI CLI Local Runtime slice.
- This is not full YonerAI cloud production.
- This is not a production network installer.
- Production Oracle/control-plane runtime is not included.
- Official Managed Cloud runtime is not included in this public repository.
- Live Discord gateway restoration is not included.
- Google production login is not enabled.
- OpenAI shared traffic is not enabled.
- Production signing keys and production trust stores are not present.
- npm and winget channels are not ready.
- SafeShell is not arbitrary shell execution.
- Workspace File Access Guard is not full file summarization.
- Persistent memory is not complete.
