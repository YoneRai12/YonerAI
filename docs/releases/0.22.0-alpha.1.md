## Summary

YonerAI v0.22.0-alpha.1 connects the public CLI to the staging Official Cloud
provider run path. A logged-in staging user can select a cloud conversation,
grant explicit provider-sharing consent, submit `run.openai_shared_text`, then
inspect run status, events, result, provider quota, capability manifest, and
module manifest.

This is staging only. It is not production Google login, not production
Official Managed Cloud, not production Oracle/cloud runtime, and not automatic
local-to-cloud upload.

## Commands

Set the staging origin when using the staging backend:

```powershell
$env:YONERAI_STAGING_AUTH_ORIGIN = "https://api-staging.yonerai.com"
```

Authenticate to staging:

```powershell
yonerai login
yonerai auth status --pretty --lang ja
```

Inspect provider gateway state:

```powershell
yonerai provider status --pretty --lang ja
yonerai provider quota --pretty --lang ja
yonerai provider models --pretty --lang ja
```

Inspect cloud conversation metadata and enable explicit provider sharing:

```powershell
yonerai sync conversations --pretty --lang ja
yonerai sync conversation show <cloud_conversation_id> --pretty --lang ja
yonerai privacy provider-sharing status <cloud_conversation_id> --pretty --lang ja
yonerai privacy provider-sharing enable <cloud_conversation_id> --sync-policy cloud_to_local --confirm --pretty --lang ja
yonerai privacy provider-sharing disable <cloud_conversation_id> --pretty --lang ja
```

Submit and inspect a staging provider run:

```powershell
yonerai run submit "hello" --capability run.openai_shared_text --conversation-id <cloud_conversation_id> --conversation-origin cloud --sync-policy cloud_to_local --provider-data-policy openai_shared_explicit --pretty --lang ja
yonerai run status <run_id> --pretty --lang ja
yonerai run events <run_id> --pretty --lang ja
yonerai run result <run_id> --pretty --lang ja
```

Inspect staging manifests:

```powershell
yonerai worker status --pretty --lang ja
yonerai capability list --pretty --lang ja
yonerai module list --pretty --lang ja
```

## What Changed Since v0.21.0-alpha.2

- Added Public CLI provider-sharing consent commands.
- Added staging provider gateway commands for status, quota, models, and disable
  report.
- Added Native Run CLI commands for submit, status, events, result, cancel,
  worker status, capability list, and module list.
- Added conversation sync policy enforcement for local-only, cloud-to-local, and
  bidirectional-explicit boundaries.
- Added public-safe Native Run response sanitization for staging provider
  payloads.
- Added AWS-backed provider context preparation before provider runs:
  - stores the selected message body on AWS
  - previews provider consent
  - approves provider consent after explicit local confirmation
  - reads context manifest metadata
  - never prints raw prompt body, provider keys, Google tokens, local paths, or
    private file bytes
- Added context preview fields for current message, prior message count,
  reserved token budget, data-sharing mode, and excluded data categories.
- Added TUI/slash command surfaces for run, worker, capability, module, provider
  sharing, and sync policy.
- Added tests for provider consent, local-only rejection, secret-like prompt
  rejection, provider gateway status/quota/model derivation, Native Run response
  sanitizer behavior, and TUI aliases.
- Added public-safe E2E evidence in
  `docs/evidence/STAGING_PROVIDER_RUN_E2E_2026_06.md`.

## Security And Privacy Boundary

- OpenAI shared traffic is off by default.
- Provider sharing requires explicit per-conversation consent.
- Selected AWS-backed conversation input and provider output may be sent to
  OpenAI when provider sharing is enabled.
- Input and output may be used under OpenAI data-sharing settings.
- Local-only conversations never call OpenAI.
- No local/workspace files or attachments are sent.
- No local private memory or local node payload is sent.
- No provider key is printed, stored, or uploaded by the public CLI.
- No Google access token, ID token, auth code, or refresh token is printed or
  stored by the public CLI.
- No automatic paid overage is enabled.
- OpenAI project budget is not treated as the hard cap; YonerAI applies its own
  hard application budget and token reservation policy.
- Ambiguous provider-result retry is not automatic.

## Negative Checks

- No provider-sharing consent: rejected before backend call.
- `local_only` plus `openai_shared_explicit`: rejected before backend call.
- Secret-like prompt: rejected before backend call.
- Consent revocation: future provider-sharing run rejected before backend call.

## Known Limitations

- This is an alpha prerelease line, not the stable CLI Local Runtime channel.
- Production Google login/passkey/account runtime is not included.
- Production AWS backend code is not included in the public repository.
- Production Oracle/cloud runtime is not included.
- Official Managed Cloud remains external/private and contract-only from the
  public repository.
- Production installer signing/trust store is not included.
- npm, winget, and auto-update apply are not included.
- `src/cogs/ora.py` remains a legacy boundary and is not solved by this release.
- `reference_clawdbot` is not touched.
