# YonerAI v2026.5.22 Web Chat MVP Review-gate Checkpoint

Date: 2026-05-22
Status: public temporary Web Chat MVP checkpoint
Scope: clarify and harden `clients/web` as a temporary local/mock chat surface
Not a production release

## Open

YonerAI public development now treats `clients/web` as a temporary Web Chat MVP and smoke-demo surface for the public Core API message contract.

This is not the final YonerAI Web product UI. It is not production readiness, not official cloud, not Google login, not persistent memory, not Discord gateway completion, and not ChatGPT-equivalent feature parity.

## Closed

- Updated `clients/web` so the main page explicitly labels itself as a temporary Web Chat MVP.
- Added mode controls for:
  - `mock` / `offline`
  - local Ollama through `local_provider: "ollama"`
  - OpenAI-compatible local servers through `local_provider: "openai_compatible_local"`
- Added model pass-through input for local mode.
- Added response metadata display for provider, mode, model, and message id.
- Added safe local LLM error display for Core API errors.
- Added loopback-only Core API rewrite origin configuration for local development when port `8001` is already occupied.
- Fixed the remaining observed `clients/web` `postcss` advisory locally with a narrow npm override and lockfile update.

## Changed

- README, README_JP, the capability matrix, and `clients/web` docs now distinguish temporary Web Chat MVP from final product UI.
- Security triage now records the remaining observed Dependabot alert as fixed in this branch pending GitHub rescan after merge.
- Open PR backlog triage now records that retired `ora-ui` PRs are no longer open and that no backlog PR is used as a base for this lane.

## Security and Hygiene

- The Web page does not expose an arbitrary provider URL input.
- The Next rewrite origin is loopback-only.
- Local provider base URL validation remains owned by the Core API loopback-only guard.
- No external OpenAI, Anthropic, Gemini, Discord, SNS, or web-search provider call is added.
- No provider API key, Discord token, Google login, private repository, deployment, production release, or memory persistence is required.
- PR review handling for this checkpoint requires all bot and human review comments to be read, classified, and either fixed or explicitly deferred before merge.
