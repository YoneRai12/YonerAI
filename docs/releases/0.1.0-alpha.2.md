# 2026.05.22 — YonerAI v0.1.0-alpha.2 Capability Slice

## Highlights

YonerAI v0.1.0-alpha.2 is a local public alpha capability slice. It gives users
a real CLI surface for mock provider execution, run IDs, workspace-scoped file
summaries, mock search, SafeShell diagnostic planning, explicit local-only
memory records, synthetic Discord gateway checks, fixture-based status output,
and Windows installer dry-run planning. It also adds opt-in external provider
adapters and loopback-only local LLM execution behind explicit safety gates. It
is not a production release, not an installer-ready release, and not a live
Discord or Official Managed Cloud release.

## What you can actually do now

These commands are intended to work without provider keys, Discord tokens,
production services, or live network calls:

- `yonerai demo --pretty`
- `yonerai doctor --pretty --lang ja`
- `yonerai status --pretty --lang ja`
- `yonerai plan "summarize public docs" --json`
- `yonerai ask "summarize public docs" --provider mock --json`
- `yonerai ask "summarize this file" --file <path> --workspace <dir> --provider mock --json`
- `yonerai search mock "YonerAI alpha2" --json`
- `yonerai ops plan git-status --json`
- `yonerai memory add "local note" --store <local.jsonl> --confirm-local --json`
- `yonerai discord synthetic "hello" --json`
- `yonerai status --source fixture --json`
- `yonerai manifest verify releases/manifest.example.json --json`
- `yonerai install plan-windows --json`

The mock `ask` path executes through the provider adapter and returns a
public-safe run result with a `run_id`. Workspace file summarization reads only
an explicitly supplied file under an explicitly supplied workspace. Memory writes
only when a local store path and `--confirm-local` are supplied.

## YonerAI CLI

- `yonerai demo` shows the current alpha2 capability slice and boundary status.
- `yonerai ask` can execute safe mock-provider tasks and returns a `run_id`.
- `yonerai ask --file ... --workspace ... --provider mock` summarizes explicit
  workspace-allowlisted text files.
- `yonerai plan` previews classification, route, provider/model decision,
  approval gates, and disabled reasons without execution.
- `yonerai search mock` returns deterministic search fixtures.
- `yonerai ops plan` exposes SafeShell diagnostic planning for a small allowlist.
- `yonerai memory` supports explicit local-only redacted memory records.
- `yonerai discord synthetic` exercises the synthetic Discord gateway boundary.
- `yonerai status --source fixture` prints public-safe official/status fixtures.
- `yonerai manifest verify` validates local manifest files only.
- `yonerai install plan-windows` builds a dry-run Windows install plan without
  downloading, installing, executing remote code, or mutating PATH.

## Provider Runtime

- Mock provider execution is deterministic and remains the default safe path.
- OpenAI-compatible execution requires explicit live execution plus
  `YONERAI_OPENAI_COMPATIBLE_API_KEY`,
  `YONERAI_OPENAI_COMPATIBLE_BASE_URL`, and
  `YONERAI_OPENAI_COMPATIBLE_LIVE=1`.
- Anthropic execution requires explicit live execution plus
  `YONERAI_ANTHROPIC_API_KEY` and `YONERAI_ANTHROPIC_LIVE=1`.
- Gemini execution requires explicit live execution plus
  `YONERAI_GEMINI_API_KEY` and `YONERAI_GEMINI_LIVE=1`.
- Local LLM execution is loopback-only and rejects non-loopback endpoints.
- Default tests and default CLI/demo paths do not make live provider calls.

## Search and Tools

- Web search is modeled as an adapter boundary.
- Mock search returns deterministic public fixtures.
- Live web search is not enabled by default.
- SafeShell is diagnostic planning only for an allowlist such as git status.
- Unknown tools, arbitrary commands, and mutation commands remain denied or
  approval-gated/plan-only.

## Local and Hybrid

- Workspace file summarization is explicit, workspace-scoped, text-only,
  size-limited, and path traversal resistant.
- Local memory v0.1 is explicit opt-in, local-only, redacted before write, and
  not enabled by default.
- Official Hybrid Private remains represented by public-safe Local Node
  contracts and fixtures.
- Official Managed Cloud remains external contract-only in this public repo.

## Discord

- A synthetic Discord gateway adapter models mention handling, progress events,
  final-once behavior, and duplicate responder prevention.
- No live Discord connection is required or enabled by default.
- This release does not restore production Discord behavior and does not use
  Discord tokens.

## Installer and Distribution

- Manifest verification is local-file validation only.
- The example manifest is contract-valid but not install-ready because it uses a
  non-production placeholder signature.
- The Windows installer planner is dry-run only.
- No remote code is downloaded or executed.
- PATH is not mutated.
- No package is installed.
- No npm package, winget package, signed production installer, production signing
  key, or production trust store is included.

## Security

- Provider errors redact configuration state and do not print API keys.
- External providers require explicit live opt-in and provider-specific env
  flags.
- Local LLM endpoints must be loopback.
- File access is explicit workspace-only and disabled unless a file/workspace is
  supplied.
- Search/tool/shell boundaries remain disabled, mock, or plan-only by default.
- Local memory is explicit local-only storage, not cloud memory.
- Official cloud and Oracle control-plane behavior remains contract/stub-only in
  the public repo.
