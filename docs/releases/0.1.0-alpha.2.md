# YonerAI v0.1.0-alpha.2 Capability Slice

## Highlights

YonerAI v0.1.0-alpha.2 moves the public alpha from a mock-only preview into a small but usable capability slice: provider execution contracts, opt-in external provider adapters, loopback-only local LLM support, workspace file summarization, mock search, SafeShell diagnostic planning, explicit local memory, synthetic Discord gateway fixtures, official status contracts, installer dry-run planning, and an expanded `yonerai demo`. It remains a public alpha prerelease, not a production release.

## YonerAI CLI

- `yonerai demo --pretty` and `yonerai demo --json` now show the alpha2 provider/runtime capability surface.
- `yonerai ask "task" --provider mock --json` executes through the mock provider and records a public-safe run result.
- `yonerai ask "summarize this file" --file <path> --workspace <dir> --provider mock --json` summarizes explicit workspace-allowlisted text files through the mock provider.
- `yonerai plan "task" --json` previews classification, route, provider/model decision, approval gates, and disabled reasons without execution.
- `yonerai search mock "query" --json` returns a deterministic mock search fixture.
- `yonerai ops plan git-status --json` previews a SafeShell diagnostic plan without arbitrary shell execution.
- `yonerai memory add "text" --store <local.jsonl> --confirm-local --json` writes an explicit local-only redacted memory record.
- `yonerai discord synthetic "message" --json` exercises the synthetic Discord gateway boundary.
- `yonerai status --source fixture --json` prints local official/status contract fixtures without contacting production services.
- `yonerai install plan-windows --json` builds a Windows installer dry-run plan without downloading, installing, or mutating PATH.

## Provider Runtime

- Mock provider execution is deterministic and remains the default safe path.
- OpenAI-compatible execution is available only when the user explicitly selects live execution and configures `YONERAI_OPENAI_COMPATIBLE_API_KEY`, `YONERAI_OPENAI_COMPATIBLE_BASE_URL`, and `YONERAI_OPENAI_COMPATIBLE_LIVE=1`.
- Anthropic execution is available only when the user explicitly selects live execution and configures `YONERAI_ANTHROPIC_API_KEY` and `YONERAI_ANTHROPIC_LIVE=1`.
- Gemini execution is available only when the user explicitly selects live execution and configures `YONERAI_GEMINI_API_KEY` and `YONERAI_GEMINI_LIVE=1`.
- Local LLM execution remains loopback-only and rejects non-loopback endpoints by design.
- Default tests and default CLI/demo paths do not make live provider calls.

## Search and Tools

- Web search is represented by a provider-like search adapter.
- Mock search returns deterministic public fixtures.
- Live web search is disabled in this alpha slice.
- SafeShell exposes diagnostic planning for a small allowlist only.
- Arbitrary shell commands, mutation commands, and unknown tools are denied or kept plan-only.

## Local and Hybrid

- Workspace file summarization is explicit, workspace-scoped, text-only, size-limited, and path traversal resistant.
- Local memory v0.1 is explicit opt-in, local-only, redacted before write, and not enabled by default.
- Official Hybrid Private remains represented by public-safe Local Node contracts and fixtures.
- Official Managed Cloud remains external contract-only in this public repository.

## Discord

- A synthetic Discord gateway adapter models mention handling, progress events, final-once behavior, and duplicate responder prevention.
- No live Discord connection is required or enabled by default.
- This release does not restore production Discord behavior and does not use Discord tokens.

## Installer and Distribution

- Manifest verification remains local-file validation only.
- The example manifest is contract-valid but not install-ready because it uses a non-production placeholder signature.
- The Windows installer planner is dry-run only. It does not download remote code, execute remote code, install packages, mutate PATH, or write production trust material.
- No npm, winget, signed production installer, or production signing key is included.

## Security

- Provider errors redact configuration state and do not print API keys.
- External providers require explicit live opt-in and provider-specific env flags.
- Local LLM endpoints must be loopback.
- File access is explicit workspace-only and disabled unless a file/workspace is supplied.
- Search/tool/shell boundaries remain disabled, mock, or plan-only by default.
- Local memory is explicit local-only storage, not cloud memory.
- Official cloud and Oracle control-plane behavior remains contract/stub-only in the public repo.

## Changed

- Expanded provider registry coverage from mock/OpenAI-compatible/local into Anthropic and Gemini opt-in adapters.
- Added workspace file summarize capability to `yonerai ask`.
- Added `yonerai search`, `yonerai ops`, `yonerai memory`, `yonerai discord synthetic`, `yonerai status --source fixture`, and `yonerai install plan-windows`.
- Expanded `yonerai demo` to show alpha2 capabilities and boundaries in one deterministic public-safe report.
- Reduced stale runtime PR debt after alpha2 implementation by closing superseded route/planning PRs.

## Known limitations

- Not production-ready.
- Official Managed Cloud is not runnable from this public repository.
- Oracle/control-plane behavior is contract/stub/local-dev only.
- Live Discord is not restored.
- Persistent memory is not complete; local memory v0.1 is explicit local-only storage.
- Live web search is not enabled.
- SafeShell is not arbitrary shell execution.
- Workspace file summarization is not arbitrary filesystem access.
- Installer distribution is not ready; only manifest verification and dry-run planning exist.
- No Google login, production DB behavior, production telemetry ingestion, npm package, winget package, production signing key, or production trust store is included.
- `src/cogs/ora.py` remains unresolved legacy/runtime boundary residue.
