# ORA 2026.2.9 Release Notes (2026-02-09)

This release switches ORA to **date-based versioning**: `YYYY.M.D`.

## Highlights (JP)
- **配布型ORAの土台**: `private/shared` プロファイル分離 + `instance_id` で状態/DB/ログ/メモリ/secret を混在させない。
- **承認(Approvals)の実体化**: 別チャネル承認 (Owner DM + `/approve`) + TTL + CRITICALコード + argsすり替え防止。
- **shared/guest規約の固定(M3)**: allowlist + unknown-riskはHIGH扱い + CRITICALデフォルト拒否で事故率を下げる。
- **Relay MVP + Harden(M2/M2.5)**: WS中継 + pairing + HTTP proxy。mux(`id -> Future`)・上限・timeout・切断掃除で外運用の壊れ方を潰した。
- **Cloudflare Quick Tunnel対応**: ドメイン無しの外部検証を `.env` で切替できる。
- **サンドボックスDL検証ツール**: GitHub repo をZIPで取得して静的スキャンのみ実施（実行はしない）。
- **使い勝手の調整**: Owner/Sharedゲストの「承認が多すぎ問題」を `.env` で制御可能に。

## Highlights (EN)
- **Distribution-ready baseline**: `private/shared` profiles + stable `instance_id` to prevent state mixing.
- **Real approvals**: out-of-band owner approval (DM + `/approve`) + TTL + CRITICAL code + anti-args-swap.
- **Shared/guest policy as code**: allowlist-first, unknown tools default to HIGH, CRITICAL blocked by default.
- **Relay MVP + hardening**: WS routing + pairing + HTTP proxy with mux, caps, timeouts, and cleanup.
- **Cloudflare Quick Tunnel**: domain-less external testing via `.env`.
- **Sandbox repo inspection tools**: download GitHub ZIP and run static inspection only (no execution).
- **QoL knobs**: reduce approval friction via `.env` while keeping safe defaults.

## Notable Env Additions
- Owner approvals:
  - `ORA_OWNER_APPROVALS` = `high|critical_only|off`
  - `ORA_OWNER_APPROVAL_SKIP_TOOLS`
  - `ORA_SHARED_OWNER_APPROVALS`, `ORA_SHARED_OWNER_APPROVAL_SKIP_TOOLS`
- Shared guest approvals:
  - `ORA_SHARED_GUEST_APPROVAL_MIN_SCORE` (default `30`, set `60` to require approvals only for HIGH+)
- LLM fallback:
  - `ORA_LLM_FALLBACK_TO_OPENAI`, `ORA_LLM_FALLBACK_MODEL`
- Sandbox limits:
  - `ORA_SANDBOX_MAX_ZIP_BYTES`

## Compatibility Notes
- Releases are now tagged as `v<VERSION>` where `VERSION` is `YYYY.M.D` (example: `v2026.2.9`).
- Existing SemVer tags remain valid for history, but new releases should follow date-based versions.
