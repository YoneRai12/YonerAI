Released: 2026-02-09

YonerAI (formerly ORA) now uses **date-based versioning**: `YYYY.M.D`.

## Highlights (JP)
- **配布型の土台が完成**: `private/shared` プロファイル + `instance_id` で DB/ログ/メモリ/secret が混ざらない。
- **承認(Approvals)が実体化**: Owner DM + `/approve` の別チャネル承認、TTL、CRITICALコード、argsすり替え防止。
- **shared/guest を“規約”として固定**: allowlist前提、未知ツールはHIGH扱い、CRITICALはデフォルト拒否。
- **Relay MVP + Hardened**: WS + pairing + HTTP proxy、mux・上限・timeout・切断掃除で外のネットでも壊れにくい。
- **ドメイン無しの外部検証**: Cloudflare Quick Tunnel を `.env` でON/OFF可能。
- **静的サンドボックス検証**: GitHub ZIPを落として“実行なし”の静的検査/比較ができる。

## Highlights (EN)
- **Distribution-ready baseline**: `private/shared` profiles + stable `instance_id` prevent state mixing.
- **Real approvals**: out-of-band owner approval (DM + `/approve`) + TTL + CRITICAL code + anti-args-swap.
- **Shared/guest policy as code**: allowlist-first, unknown tools default to HIGH, CRITICAL blocked by default.
- **Relay MVP + hardening**: WS routing + pairing + HTTP proxy with mux, caps, timeouts, and cleanup.
- **Cloudflare Quick Tunnel**: domain-less external testing via `.env`.
- **Sandbox repo inspection tools**: download GitHub ZIP and run static inspection only (no execution).

## New In This Release Drop (Discord QoL)
- **“普通に使える”Discord機能をゲストにも**:
  - デフォルトの公開ツールに VC/TTS/音楽/安全検索 を追加（ただし `ORA_SHARED_GUEST_ALLOWED_TOOLS` を設定している場合はそちらが優先）。
- **安全なWEB検索ツール `web_search_api`**:
  - SerpApi / DuckDuckGo で検索（ブラウザ自動操作なし、ダウンロードなし）。
- **音楽再生が現実に使えるUXへ**:
  - `@Bot 〇〇 流して` で YouTube検索して再生。
  - URL貼り付け、音声ファイル(mp3等)添付も再生対応。
  - **DiscordネイティブのスクロールUI(Selectメニュー)**で候補から選んで再生可能。
  - **プレイリスト丸ごとキュー追加（@メンション + URL）**:
    - YouTube: `list=...` のURLなら全曲をバックグラウンドで解決してキュー投入。
    - Spotify: プレイリスト/アルバムURLから曲名を取得し、YouTube検索で再生（Spotify音源は直接再生しない）。

  - `ORA_MUSIC_MENTION_TRIGGERS=1`
  - `ORA_MUSIC_MENTION_LEVEL=owner|vc_admin|user`
  - `ORA_MUSIC_MENTION_YOUTUBE_LEVEL=vc_admin|user`

  - `ORA_MUSIC_NATIVE_PICKER=1`

- Approvals knobs:
  - `ORA_OWNER_APPROVALS`, `ORA_OWNER_APPROVAL_SKIP_TOOLS`
  - `ORA_SHARED_GUEST_APPROVAL_MIN_SCORE`

## Compatibility Notes
- Tags are `v<VERSION>` where `VERSION` is `YYYY.M.D` (example: `v2026.2.9`).
- SemVer tags remain in history, but new releases follow DateVer.
