# v2026.5.20.12 Local LLM Error Reporting Hardening Checkpoint

Status: public local LLM error/reporting checkpoint. Not production, not deploy,
and not provider ecosystem completion.

## Open

- Local LLM model discovery remains optional and not implemented here.
- Broader provider boundary hardening remains a separate lane.
- Final Web product UI, Google login, persistent memory, official cloud,
  Discord gateway, web search, and external provider live generation remain open.

## Closed

- Public local LLM errors now return safe `mode`, `provider`, `model`, and
  `status` metadata when available.
- Surface API run errors preserve the same safe local provider context.
- CLI HTTP error output now preserves useful local provider context without
  printing secret-like values or local machine paths.
- CLI URL failures no longer print raw exception reasons that may include local
  paths or hostnames.
- Secret-like local model names are rejected before reaching the local provider.

## Changed

- `POST /v1/public/messages` local-mode errors remain deterministic and
  public-safe while becoming easier to debug.
- `POST /api/v1/agent/run` mirrors public message error metadata for local mode.
- The local smoke CLI formats safe error context consistently for public Core
  API failures.
- README and capability matrix now point to this checkpoint.

## Security and Hygiene

- Local provider URLs remain loopback-only.
- Remote, LAN, tunnel, credential-bearing, query-string, and fragment-bearing
  local provider URLs remain rejected.
- Error responses must not expose Authorization headers, token-like values, local
  absolute paths, usernames, hostnames, raw chain-of-thought, or private runtime
  details.
- No provider API key path was added.

## Not Included

- External OpenAI / Anthropic / Gemini live generation.
- Provider ecosystem completion.
- Production deployment.
- Production auth.
- Persistent memory.
- Google login.
- Final Web UI.
- Discord gateway completion.
- Control-plane or private runtime ingestion.

## Still Open

- Capability / extension boundary hardening.
- Tools/MCP safe subset contract.
- Security PR backlog resolution for stale access PRs.
- Local LLM model listing and richer troubleshooting docs.

## Next

Continue with capability / extension boundary hardening, keeping unknown
capabilities deny-by-default and tools/MCP disabled unless explicitly safe.
