Tag: `v0.7.0`

Release type: stable CLI Local Runtime release.

## Summary

v0.7.0 promotes the local Memory UX and Agent Console surfaces into the stable
YonerAI CLI Local Runtime lane. Users can install the stable CLI, run `yonerai`,
open Japanese-first slash-command screens, manage local memory boundaries, see
task/agent state, and inspect update/install status without enabling production
cloud, production Oracle, live Discord, Google production login, or live
external providers by default.

## What Changed Since v0.6.5

- Promoted the Memory Boundary Runtime UX from alpha into the stable CLI lane.
- Added stable `yonerai memory status/add/list/forget/sync preview` flows that
  use a local memory store by default.
- Connected memory IDs, not raw memory content, into `ask --auto` and the run
  ledger.
- Promoted the Agent Console command palette, mode, plan, review, and subagent
  preview surfaces into the stable CLI lane.
- Preserved the v0.6.5 repair-safe installer behavior and updated the stable
  install target to v0.7.0.
- Kept production cloud, production Oracle, production Google login, live
  Discord, production installer signing/trust, npm/winget, and auto-update
  apply out of scope.

## Memory Boundary

Memory in this release is local and boundary-first. A local memory record has an
ID, scope, sensitivity, sync policy, redacted summary, source reference, and
audit reason. The stable default is local-only memory. Local-to-cloud sync is
disabled by default, local_private and secret_like memory cannot sync, and
cloud-to-local remains preview/contract behavior only in the public repo.

## Agent Console

The interactive CLI now exposes a safer Codex-style agent console surface:
planner, researcher, reviewer, implementer, and tester are visible as planned
roles, not uncontrolled autonomous workers. The console can show plans, review
summaries, blocked/approval_required states, route/provider decisions, run_id,
and memory_used IDs without exposing raw chain-of-thought.

## Security And Boundary

- Provider keys are not printed or stored by default.
- External providers remain opt-in and require explicit live/provider flags.
- Local LLM remains loopback-only.
- Workspace file access remains explicit and workspace-scoped.
- No arbitrary shell/file/tool execution is enabled.
- `yonerai.com` remains a command/wrapper entry. GitHub Release assets remain
  the install source of truth.
- `install.ps1` remains verified by SHA256 sidecar before execution in the
  short install path.

## Known Limitations

- Not full YonerAI production readiness.
- Official Managed Cloud is external/contract-only in the public repo.
- Production Oracle/cloud runtime is not included.
- Live Discord is not restored.
- Production Google login/passkeys are not implemented.
- Production signing keys and production trust store are not included.
- npm and winget channels are not ready.
- Auto-update apply and forced silent update are not implemented.
- Persistent cloud memory is not complete.
- Local/private memory is not uploaded automatically.
- `src/cogs/ora.py` is not solved.
