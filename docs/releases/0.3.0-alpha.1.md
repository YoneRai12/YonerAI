# 2026.05.26 — YonerAI v0.3.0-alpha.1 Interactive CLI Slice

Release date: 2026-05-26
Tag: `v0.3.0-alpha.1`
Status: alpha pre-release
Previous pre-release: `v0.2.0-alpha.1`

## Summary

YonerAI v0.3.0-alpha.1 adds a Japanese-first interactive terminal shell on top
of the existing safe `ask --auto` runtime. Users can start with `yonerai`, chat,
open settings, inspect providers, inspect safety boundaries, and view recent
runs without reading JSON. This is still an alpha CLI slice, not production
Oracle, Official Managed Cloud, live Discord, a production installer, or an
arbitrary shell/tool runner.

## What you can actually run now

| Command | What it does | What it does not do | Mode |
| --- | --- | --- | --- |
| `yonerai` | Starts the interactive CLI when stdin is a TTY. On first launch it asks for Japanese or English. | Does not start a GUI, deploy services, call live providers by default, or store secrets. | Interactive local |
| `yonerai chat` | Starts the same interactive CLI explicitly. | Does not require provider keys or live services. | Interactive local |
| `yonerai chat --script --lang ja` | Reads chat/script lines from stdin for tests or automation without hanging on non-TTY input. | Does not become a shell runner and does not execute arbitrary commands. | Scripted local |
| `yonerai config show --pretty --lang ja` | Shows local CLI preferences such as language/provider/approval/file-access mode. | Does not print secrets, provider keys, or local absolute config paths. | Local config |
| `yonerai config set language ja` | Stores the display language preference locally. | Does not store API keys, tokens, prompts, completions, or provider credentials. | Local config |
| `/設定` | Shows language, provider preference, local LLM state, approval mode, file access mode, live connection state, and boundary notes in Japanese. | Does not enable live network/provider execution by itself. | Interactive slash command |
| `/提供元` | Shows provider readiness with Japanese labels and redacted key handling. | Does not print keys or call providers. | Interactive slash command |
| `/安全` | Shows network, file access, and tools policy in Japanese. | Does not grant shell, network, or arbitrary file access. | Interactive slash command |
| `/履歴` | Lists recent redacted runs from the configured local ledger. | Does not upload run history or show raw secrets/local paths. | Interactive slash command |
| `/表示 <実行ID>` | Shows one redacted run by run_id. | Does not expose raw chain-of-thought, provider keys, or arbitrary local paths. | Interactive slash command |
| `/提供元選択 モック` | Changes the interactive provider preference to the mock provider. | Does not call external providers and does not make mock output production-ready. | Interactive slash command |
| `/終了` | Exits the interactive CLI. | Does not mutate runtime state beyond saved non-secret preferences. | Interactive slash command |
| `/settings` | Compatibility alias for `/設定`, accepted even in Japanese mode. | It is accepted for compatibility, but Japanese mode still displays Japanese-first labels. | Interactive alias |
| `yonerai ask "hello" --auto --pretty --lang ja` | Still works outside the interactive shell and shows route/provider/run_id details. | Does not use live providers by default. | CLI runtime |
| `yonerai providers --pretty --lang ja` | Still shows provider readiness outside the interactive shell. | Does not print keys or run live calls. | Offline provider readiness |
| `yonerai runs list --ledger <local.jsonl> --pretty --lang ja` | Still shows redacted local run history outside the interactive shell. | Does not upload or cloud-sync history. | Local-only ledger |

## Japanese interactive commands

Japanese mode displays these commands first:

```text
/設定
/提供元
/安全
/履歴
/表示 <実行ID>
/言語 日本語|英語
/提供元選択 自動|モック|ローカル|オープンAI互換|アンソロピック|ジェミニ
/承認 確認|拒否
/ファイル ワークスペース内のみ|無効
/終了
```

English slash commands such as `/settings`, `/providers`, `/safety`, `/runs`,
`/provider mock`, and `/quit` remain accepted as compatibility aliases even
when the display language is Japanese.

## What changed since v0.2.0-alpha.1

- Added `yonerai` / `yonerai chat` as a standard-library interactive terminal
  shell.
- Fixed release-blocking review debt from prior merged PRs: cloud-contract route
  metadata now preserves public-reasoning classification, CLI package versions
  normalize PEP 440 prereleases back to public SemVer, and interactive/config
  setting errors are controlled.
- Hardened final release-gate review blockers: interactive setting write errors
  no longer look like invalid input, Oracle stub queue refuses private/local
  path tasks, deployment word variants remain disabled, relay `auto` URL status
  resolves the URL file before trusting loopback state, and extension manifest
  public fields redact unknown capability/version input.
- Fixed post-merge P2 review debt before release: unknown extension capability
  inputs remain distinct until evaluation while public output stays redacted,
  and relay pretty output now treats `auto_resolved_loopback` as a safe
  loopback state.
- Added first-launch language selection and safe local CLI config.
- Added Japanese-first slash command surfaces for settings, provider readiness,
  safety policy, run history, and run display.
- Connected chat input to the existing `ask --auto` path so responses show
  run_id, route, provider, approval state, and output.
- Added non-TTY fallback and scripted mode so pipes/CI do not hang.
- Updated README/CLI docs with the new first-run path and alpha boundaries.

## Provider runtime

- Mock provider remains the default and requires no key.
- Local LLM execution remains explicit and loopback-only through existing
  provider runtime controls.
- OpenAI-compatible, Anthropic, and Gemini live calls remain opt-in only and
  require explicit provider configuration plus `--live`.
- The interactive shell does not store provider keys.

## Safety and boundaries

- Network is off by default.
- Tools remain dry-run/diagnostic only; this release does not add arbitrary
  shell execution.
- File access remains workspace-scoped and explicit. This release does not add
  arbitrary file access, folder crawling, PDF/image parsing, or real document
  summarization.
- Run history remains redacted and local-only when a ledger is explicitly used.
- Non-TTY input prints guidance instead of blocking for interactive input.

## Known limitations

- Not production-ready.
- Official Managed Cloud is not runnable from this public repository.
- Oracle/control-plane behavior remains local-dev stub/contract work.
- Live Discord is not restored.
- Persistent memory is not complete.
- Live web search is not default/live.
- SafeShell is not arbitrary shell.
- Workspace file behavior is an access guard only.
- Production installer, npm, and winget are not ready.
- No Google login.
- No production DB behavior.
- No production telemetry ingestion.
- `src/cogs/ora.py` remains a legacy boundary and is not claimed solved.
