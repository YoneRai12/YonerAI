## Summary

YonerAI v0.8.0 promotes the current trunk into a normal, talkable local CLI
runtime. A new user can install, type `yonerai`, choose Japanese or English,
type a normal message, open `/`, and use short commands without memorizing long
flag sets.

This release is stable for the local CLI runtime slice only. It is not full
YonerAI cloud production, not production Google login, not production Oracle,
not live Discord, and not an automatic local-to-cloud upload system.

## First 3 Minutes

Install:

```powershell
irm https://install.yonerai.com | iex
```

Start:

```powershell
yonerai
```

Then choose `日本語` or `English` and type a normal message. The first offline
answer uses the safe local/mock path. Useful slash commands:

```text
/設定
/更新
/ログイン
/認証
/セッション
/プロジェクト
/API
/レート
/同期
/記憶
/履歴
/ヘルプ
/終了
```

Short shell commands:

```powershell
yonerai
yonerai chat
yonerai update
yonerai update stable
yonerai update beta
yonerai login
yonerai whoami
yonerai sessions
yonerai projects
yonerai ping
yonerai rate-limit
```

Advanced/CI flags remain available, but normal users should not need:

```powershell
yonerai auth google login --staging --bridge --open-browser --wait-linked --pretty --lang ja
```

Use `yonerai login` instead when staging is configured.

## What Changed Since v0.7.0

### Normal Interactive CLI

- `yonerai` now opens the interactive app directly in a TTY.
- `yonerai chat` remains an alias for the same app.
- Plain text in the composer goes through the safe `ask --auto` path.
- The first answer works offline with the mock provider and no provider key.
- The home screen is more compact: version/channel, language, provider,
  memory, update, auth, safety, and suggested next actions.
- First launch persists language and does not block local use on login.
- Japanese UI uses Japanese labels first; English aliases remain available.
- Command display modes support Japanese-only, Japanese with English aliases,
  English with Japanese aliases, and English-only display.
- Sub/secondary command labels are dimmed where the renderer supports it.

### Command Palette And Short Commands

- `/` opens the command palette in the interactive app.
- Short commands were promoted for normal users:
  `yonerai login`, `yonerai whoami`, `yonerai sessions`, `yonerai revoke`,
  `yonerai logout`, `yonerai projects`, `yonerai ping`, and
  `yonerai rate-limit`.
- Every short shell command has a slash twin for the app:
  `/ログイン`, `/認証`, `/セッション`, `/プロジェクト`, `/API`, `/同期`,
  `/記憶`, `/設定`, `/レート`, `/監査`, `/方針`, `/更新`, `/ヘルプ`, and
  `/終了`.
- Long staging flags are kept for advanced/CI use, not the normal path.

### Update And Install

- `yonerai update` shows a two-step choice: `安定版` or `ベータ版`.
- `yonerai update beta` is the user-facing prerelease command. Internally it
  maps to the existing prerelease manifest channel for compatibility.
- `/更新` shows current version, stable/beta choices, artifact/hash status, and
  the next safe command.
- `/更新 適用 ...` requires explicit confirmation. There is no silent update.
- Update apply is guarded and testable without real install through a test mode.
- `install.ps1` supports an opt-in `-Shortcut` flag for Start Menu/Desktop
  shortcuts. It does not require admin rights and does not mutate PATH by
  default.
- `install.yonerai.com` and README install text now point at v0.8.0 stable.

### Memory And Agent Console

- Local memory runtime from the v0.12 prerelease line is now part of the stable
  CLI Local Runtime experience.
- `/記憶` and `/メモリ` expose local memory status, add/list/forget, and sync
  preview boundaries.
- Memory ledger usage stores IDs only, not raw memory content.
- Local-to-cloud memory/conversation upload remains disabled by default.
- Agent console features from the v0.13 prerelease line are available as
  public-safe previews: `/計画`, `/レビュー`, `/エージェント`, `@planner`,
  `@reviewer`, and `@researcher`.
- These previews do not start uncontrolled autonomous agents.

### Official/Cloud Boundary Visibility

- Staging Google login UX from v0.17/v0.18 is visible through short commands
  and `/認証`, but production Google login remains unavailable.
- Staging cloud conversation sync preview from v0.19 is visible through
  `/同期` and short sync commands, but local-to-cloud upload remains explicit
  and disabled by default.
- Control Spine client UX from v0.20/v0.21 is visible through `whoami`,
  `sessions`, `projects`, `ping`, `rate-limit`, and related TUI screens.
- Status/API contract and rate-limit display remain public-safe. Private
  endpoints, private IPs, local paths, AWS ARNs, and secret-like markers are
  rejected or redacted before public output.

### Security And Policy

- Token-custody regression is a standing gate: the CLI must not persist Google
  tokens, auth codes, or refresh tokens.
- Session expiry shows guided relogin instead of stack traces.
- Contract skew warnings read API version/min CLI version when present and
  degrade silently when fields are missing.
- `agent:run` and `admin:*` stay frozen and require a separate threat-model
  gate before use.
- Policy runtime shows provider/model/pricing/permission/runtime/update/memory
  boundaries through `yonerai policy status` and `/方針`.
- Dangerous capabilities remain deny-by-default.
- No provider keys are printed or stored.
- OpenAI shared traffic remains disabled.

## Not Included

- Full Official Managed Cloud production runtime.
- Production Google login/passkeys/accounts.
- Production Oracle/cloud runtime.
- Live Discord runtime.
- OpenAI shared traffic.
- Automatic local-to-cloud private upload.
- Arbitrary shell/file/tool execution.
- Production signing key or production trust store.
- npm/winget installer.
- Broad `src/cogs/ora.py` solution.
- Any modification to `reference_clawdbot`.
