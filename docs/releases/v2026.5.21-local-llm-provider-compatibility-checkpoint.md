# YonerAI v2026.5.21 Local LLM Provider Compatibility Checkpoint

Date: 2026-05-21
Status: public local LLM provider compatibility checkpoint
Scope: loopback-only Ollama and OpenAI-compatible local provider layer
Not a production release

## Open

YonerAI public development now treats local LLM support as a provider boundary instead of an Ollama-only special case.

This checkpoint is still local, loopback-only, and public-safe. It is not production readiness, not official cloud, not Google login, not persistent memory, not Discord gateway completion, and not ChatGPT-equivalent feature parity.

## Closed

- Kept the existing Ollama-compatible `/api/chat` path.
- Added an OpenAI-compatible local `/v1/chat/completions` adapter path.
- Added provider selection through `local_provider`:
  - `ollama`
  - `openai_compatible_local`
- Preserved backwards compatibility: omitted `local_provider` still defaults to Ollama.
- Added tests for:
  - provider normalization and documented local aliases
  - OpenAI-compatible request shape
  - OpenAI-compatible response parsing
  - model name pass-through
  - loopback-only URL rejection for remote provider URLs
  - endpoint-level provider selection and unsupported-provider rejection

## Changed

- README, README_JP, and the capability matrix now describe local provider compatibility.
- `.env.example` documents `ORA_LOCAL_LLM_PROVIDER`, optional temperature, optional max tokens, and local OpenAI-compatible base URL examples.
- The optional local LLM smoke script can target either `ollama` or `openai_compatible_local`.
- The v7.7 implementation addendum records this as an implementation expansion, not a v7.8 design replacement.

## Security and Hygiene

- Local provider URLs remain loopback-only.
- Allowed loopback hosts are `localhost`, `127.0.0.1`, and `::1`.
- Arbitrary remote URLs, LAN hosts, `0.0.0.0`, file URLs, embedded credentials, query strings, fragments, provider APIs, tunnels, and control-plane endpoints remain rejected by default.
- No external OpenAI, Anthropic, Gemini, Discord, SNS, or web-search provider call is added.
- No provider API key, Discord token, Google login, private repository, deployment, or production release is required.
- Dependabot and open PR backlog state remains visible; this checkpoint does not claim broad vulnerability remediation.
