# 2026.06.01 - YonerAI v0.13.0-alpha.1 Agent Console Runtime

Release date: 2026-06-01
Tag: `v0.13.0-alpha.1`
Status: prerelease for agent-console interaction, command palette, modes, approval profiles, and public-safe subagent previews
Previous release: `v0.12.0-alpha.2`

## Summary

YonerAI v0.13.0-alpha.1 adds a more Codex/opencode-like interactive CLI layer on top of the memory UX shipped in v0.12.0-alpha.2. The interactive app now exposes a command palette, agent modes, approval profiles, planning/review screens, and public-safe `@planner`, `@reviewer`, and `@researcher` previews. These features help users see what YonerAI intends to do before any runtime path is selected.

This release does not enable uncontrolled autonomous subagents, arbitrary shell/tool execution, production cloud memory, production Google login, live Discord, OpenAI shared traffic, production Oracle/control-plane runtime, production signing/trust, npm/winget, or a production network installer.

## Agent modes

- `plan_readonly`: default planning/read-only style for safe local previews.
- `build_safe`: a label for safe-build intent. It does not bypass public runtime tool boundaries.
- `review`: reviewer-oriented display and approval posture.
- `memory`: memory-oriented display and local memory workflow.

Mode changes are local configuration changes only. They do not grant production cloud access, shell access, arbitrary file access, provider-key access, or background autonomous execution.

## Approval profiles

- `read_only`: strict read-only posture.
- `dry_run_only`: plan-only posture with `approval_mode=prompt`; switching from read-only back to dry-run-only clears the stricter deny approval state.
- `auto_safe`: safe-build label for bounded local behavior.
- `ask_before_risky`: review/approval posture for risky requests.

The public runtime remains deny-by-default for arbitrary shell, files outside workspace boundaries, live provider calls, and production cloud/oracle paths.

## Public-safe subagent previews

The `@planner`, `@reviewer`, and `@researcher` commands are display previews, not uncontrolled autonomous subagents. They show the intended role, task framing, and boundary reminders. They do not spawn workers, browse the web, execute shell commands, access arbitrary files, push GitHub changes, create releases, deploy, or call live providers by default.

## Task progress and memory visibility

- Existing `ask --auto` progress still records classification, route, provider selection, execution/review/result status, and run ids through the public runtime paths.
- The interactive console exposes mode, permissions, route/provider context, memory status, update state, and run/history surfaces.
- Memory references are shown as `memory_used` ids only. Raw memory content, provider keys, local absolute paths, raw chain-of-thought, and private sync payloads are not placed in ledger output.

## Known limitations

- This is a prerelease.
- It is not full YonerAI production.
- Production cloud memory is not implemented.
- Production Oracle/control-plane runtime is not included.
- Official Managed Cloud runtime is not included in this public repository.
- Production Google login/passkey/account system is not enabled.
- Live Discord gateway restoration is not included.
- OpenAI shared traffic is not enabled.
- npm and winget channels are not available.
- Local private memory sync up is not automatic.
- SafeShell is not arbitrary shell execution.
- Workspace File Access Guard is not folder crawling, PDF/image parsing, arbitrary file access, or full file summarization.
- Persistent memory remains an alpha boundary runtime, not a complete product memory system.
- Self-evolution does not mutate code, GitHub, releases, deployments, or production configuration.
- `src/cogs/ora.py` is not solved by this release.
- `reference_clawdbot` is not touched.
