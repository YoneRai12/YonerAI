# 2026.06.01 - YonerAI v0.10.0-alpha.1 Public Orchestration Boundary

Release date: 2026-06-01
Tag: `v0.10.0-alpha.1`
Status: prerelease for public orchestration, TUI status, install/site alignment, and safety gates
Previous release: `v0.9.0-alpha.1`

## Summary

YonerAI v0.10.0-alpha.1 collects the public work that landed after the v0.9
TUI value-completion prerelease. It makes the Japanese-first Mission Control CLI
easier to orient with `/状態` and `/ホーム`, keeps provider/local LLM setup
visible without enabling live providers by default, hardens the public Google
OAuth boundary to dry-run contract responses, strengthens Quality Wall scanning,
and aligns the plan-only installer/site content with the current prerelease.

This is still a prerelease for the public local/runtime boundary. It is not full
YonerAI production. It does not enable production cloud, production Oracle,
production Google login, live Discord, OpenAI shared traffic runtime, production
signing/trust, npm/winget, arbitrary shell/file/tool execution, or a production
network installer.

## What changed since v0.9.0-alpha.1

- Added Japanese `/状態` and `/ホーム` entries for Mission Control status and
  navigation.
- Improved model/provider completion surfaces and kept Japanese-first labels in
  the interactive TUI.
- Fixed public Google OAuth web routes so the public repo remains dry-run /
  contract-only instead of attempting production OAuth.
- Strengthened Quality Wall scans for secret/local-path/hidden-Unicode checks
  and release-gate validation across PR and push ranges.
- Added v0.9/v0.10 yonerai.com release/press/install content foundations.
- Moved the plan-only `install.ps1` default manifest/artifact to
  `0.10.0-alpha.1`.

## Provider and local LLM boundary

- Mock provider remains safe by default.
- Local LLM remains loopback-only.
- OpenAI-compatible, Anthropic, and Gemini remain explicit live/env opt-in
  paths or setup/status surfaces.
- Provider keys are not printed or stored by default.
- Default CI remains mock/local-fixture only.

## Hybrid, official, and private boundary

- Hybrid Wire, Local Node, Relay, and Oracle surfaces remain public-safe
  local-dev contracts/stubs.
- Official Managed Cloud remains external/contract-only in this public repo.
- Private file content must not be routed to cloud_contract or Oracle stubs.
- Production Oracle/control-plane runtime is not included.
- Private handoff details, live routes, break-glass details, and production
  runtime inventory are not exposed.

## Security and boundary

- No production Oracle/cloud runtime.
- No Official Managed Cloud runtime in this public repository.
- No production Google login.
- No OpenAI shared traffic runtime.
- No live Discord or Discord token use.
- No deploy/public tunnel.
- No arbitrary shell/file/tool execution.
- No provider live call by default.
- No provider key output or config/ledger storage.
- Local LLM remains loopback-only.
- `install.ps1` remains plan-only.
- Production signing keys and production trust stores are not included.
- Self-evolution remains proposal-only.
- `src/cogs/ora.py` is not solved by this release.
- `reference_clawdbot` is not touched.

## Known limitations

- This is a prerelease, not a stable YonerAI cloud release.
- It is not full YonerAI production.
- It is not a production network installer.
- Production Oracle/control-plane runtime is not included.
- Official Managed Cloud runtime is not included in this public repository.
- Live Discord gateway restoration is not included.
- Google production login is not enabled.
- OpenAI shared traffic is not enabled.
- npm and winget channels are not available.
- SafeShell is not arbitrary shell execution.
- Workspace File Access Guard is not full file summarization.
- Persistent memory is not complete.
- Self-evolution does not mutate code, GitHub, releases, deployments, or
  production configuration.
