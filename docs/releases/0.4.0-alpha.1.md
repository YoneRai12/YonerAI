# 2026.05.26 — YonerAI v0.4.0-alpha.1 Mission Control CLI Slice

Release date: 2026-05-26
Tag: `v0.4.0-alpha.1`
Status: alpha pre-release
Previous pre-release: `v0.3.0-alpha.1`

## Summary

YonerAI v0.4.0-alpha.1 turns the interactive CLI into a Mission Control view:
users can see provider choice, route, local node state, ledger state, safety
mode, run_id, task progress, and reviewer/subagent planning without reading raw
JSON. This is still a local alpha CLI slice. It is not production Oracle,
Official Managed Cloud, live Discord, a production installer, or an arbitrary
shell/tool runner.

## Mission Control behavior

- The startup header shows provider, route, local node state, ledger state,
  safety mode, live-provider state, and setup state.
- Each `ask --auto` run now includes a task timeline:
  `classify`, `route`, `provider_selection`, `execution`, `review`, and
  `result`.
- Timeline states are deterministic: `pending`, `running`, `done`, `skipped`,
  `blocked`, and `error`. Current one-shot execution usually records completed,
  skipped, blocked, or error states after the run finishes.
- Interactive output and `runs show` display the same redacted plan/history
  information. JSON keeps machine-readable details for scripts.

## Subagent and reviewer plan

YonerAI now exposes a public-safe plan model with roles for planning, research,
implementation, testing, and review. This is visibility only. It does not launch
uncontrolled subagents, live Discord workers, external providers, or shell tools.

## Security and review fixes

- Interactive pretty output now escapes terminal control characters such as
  ANSI/OSC/BEL as `\xNN` text before printing user-controlled values.
- Image follow-up context now skips the current persisted user turn before
  looking for the latest prior user image, then stops at the latest prior user
  turn if it has no image. This prevents stale screenshot carryover.
- Image follow-up turns no longer receive both the broad image output contract
  and the follow-up contract at the same time.

## What changed since v0.3.0-alpha.1

- Improved the local CLI startup/runtime surface so `yonerai`, `yonerai chat`,
  `yonerai ask --auto`, and `yonerai runs` explain runtime state more plainly.
- Added Mission Control status, task progress, and reviewer/subagent plan
  visibility to the interactive CLI.
- Added task_progress JSON and redacted ledger events to `ask --auto`.
- Closed current-main security/review debt for terminal control characters and
  image follow-up context handling.

## Known limitations

- Not production-ready.
- Official Managed Cloud is not runnable from this public repo.
- Production Oracle/control-plane is not implemented.
- Live Discord is not restored.
- External providers still require explicit `--live` plus provider-specific
  opt-in/configuration.
- Local LLM remains loopback-only.
- SafeShell/tool execution remains dry-run/deny-by-default; arbitrary shell is
  not included.
- Workspace file access remains allowlist/guard behavior, not arbitrary file
  access or full document parsing.
- Persistent memory is not complete.
- Production installer, npm, and winget distribution are not ready.
- `src/cogs/ora.py` is not solved or broadly refactored.
