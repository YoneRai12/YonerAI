# 2026.05.31 - YonerAI v0.9.0-alpha.1 TUI Value Completion and Quality Wall

Release date: 2026-05-31
Tag: `v0.9.0-alpha.1`
Status: prerelease for Japanese-first TUI usability and public safety gates
Previous release: `v0.8.0-alpha.1`

## Summary

YonerAI v0.9.0-alpha.1 improves the local CLI experience after the
install/auth boundary alpha. The interactive TUI now gives Japanese-first
slash-command value candidates for settings such as provider, language, file
access, ledger, live connection, network, and update notices. This release also
hardens the public CI/release gate and boundary checks introduced after v0.8.

This is not full YonerAI production. It does not enable production cloud,
production Oracle, production Google login, live Discord, production
signing/trust, npm/winget, arbitrary shell/file/tool execution, or a production
network installer.

## What changed since v0.8.0-alpha.1

- Added context-aware slash-command value completion in the interactive TUI.
- Added Japanese-visible setting commands for language, provider choice,
  approval mode, workspace file access, local ledger, live-provider mode,
  network mode, update notice, local LLM guidance, and numbered selection.
- Kept English slash aliases compatible while keeping Japanese mode visually
  Japanese-first.
- Fixed provider value completion so Japanese-mode provider candidates are
  accepted by the existing value canonicalization path.
- Avoided rebuilding the slash-command alias map on every completion keystroke.
- Strengthened quality-wall and release-gate checks after v0.8 so future
  changes are harder to release with unsafe boundaries or overclaims.

## Security and boundary

- No production Oracle/cloud runtime.
- No Official Managed Cloud runtime in this public repository.
- No production Google login.
- No OpenAI shared traffic runtime.
- No live Discord or Discord token use.
- No deploy/public tunnel.
- No arbitrary shell/file/tool execution.
- No provider live call by default.
- No provider key output or config/ledger storage.
- Local LLM remains loopback-only.
- `install.ps1` remains plan-only.
- Production signing keys and production trust stores are not included.
- `src/cogs/ora.py` is not solved by this release.
- `reference_clawdbot` is not touched.

## Known limitations

- This is a prerelease, not a stable YonerAI cloud release.
- It is not full YonerAI production.
- It is not a production network installer.
- Production Oracle/control-plane runtime is not included.
- Official Managed Cloud runtime is not included in this public repository.
- Live Discord gateway restoration is not included.
- Google production login is not enabled.
- OpenAI shared traffic is not enabled.
- npm and winget channels are not available.
- SafeShell is not arbitrary shell execution.
- Workspace File Access Guard is not full file summarization.
- Persistent memory is not complete.
- Self-evolution is proposal-only and does not mutate code, GitHub, releases,
  deployments, or production configuration.
