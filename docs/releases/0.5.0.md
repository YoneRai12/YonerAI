# 2026.05.26 - YonerAI CLI Local Runtime v0.5.0

Release date: 2026-05-26
Tag: `v0.5.0`
Status: stable for the local CLI runtime slice
Previous pre-release: `v0.4.0-alpha.1`

## Summary

YonerAI CLI Local Runtime v0.5.0 is the first non-prerelease release for the
local CLI application surface. It is stable for install-and-run local CLI usage:
users can install the CLI package from this checkout, type `yonerai`, choose a
language, chat with the safe default mock provider, open settings, inspect task
progress, see reviewer/subagent planning, review safety boundaries, and view
redacted run history. This is not full YonerAI cloud production.

## What users can run

| Command | What it does | What it does not do | Mode |
| --- | --- | --- | --- |
| `yonerai` | Starts the interactive Mission Control CLI when stdin is a TTY. In a non-TTY pipe it prints fallback instructions instead of hanging. | Does not start a GUI, deploy services, call live providers by default, or store secrets. | Local CLI |
| `yonerai chat` | Starts the same interactive CLI explicitly. | Does not require provider keys or live services. | Local CLI |
| `yonerai chat --script --lang ja` | Runs the chat path from scripted input for tests or automation. | Does not execute arbitrary shell commands. | Scripted local |
| `yonerai config show --pretty --lang ja` | Shows local non-secret preferences such as language, provider preference, safety, ledger, and live-provider mode. | Does not print or store provider API keys. | Local config |
| `yonerai config set language ja` | Sets Japanese as the local UI language. | Does not write secrets or cloud config. | Local config |
| `yonerai providers --pretty --lang ja` | Shows mock, local LLM, OpenAI-compatible, Anthropic, and Gemini readiness plus capability negotiation. | Does not make live provider calls or print keys. | Local status |
| `yonerai ask "hello" --auto --pretty --lang ja` | Classifies the task, chooses a safe route, executes the default mock path, returns a run_id, and shows progress. | Does not call live providers by default. | Local runtime |
| `yonerai ask "hello" --provider local --live --json` | Uses a loopback-only local LLM endpoint when explicitly configured. | Rejects non-loopback URLs and does not use external provider keys. | Opt-in local live |
| `yonerai runs list --ledger <local.jsonl> --pretty --lang ja` | Lists redacted local runs from an explicitly selected ledger. | Does not cloud-sync history or expose raw secrets/local absolute paths. | Local-only ledger |
| `yonerai runs show <run_id> --ledger <local.jsonl> --pretty --lang ja` | Shows route, provider, progress, reviewer/subagent plan, and result summary. | Does not expose raw chain-of-thought. | Local-only ledger |
| `yonerai doctor --pretty --lang ja` | Explains local setup state, provider readiness, manifest status, and safety boundaries in plain Japanese. | Does not install packages, mutate PATH, or connect to production services. | Offline diagnostic |
| `yonerai demo --pretty` | Shows the public local capability slice without credentials. | Does not require Core API, provider keys, or live Discord. | Offline demo |

## Interactive CLI behavior

- First launch asks for `日本語` or `English` and stores only local non-secret
  preferences.
- Japanese mode shows Japanese-first slash commands and labels. English aliases
  such as `/settings`, `/providers`, `/safety`, `/tasks`, `/agents`, `/runs`, and
  `/show <run_id>` remain accepted.
- The header shows provider, model/local LLM state, route, safety mode, ledger
  state, Local Node state, and Oracle stub state.
- Settings expose language, provider preference, local LLM status, approval
  mode, safety mode, ledger mode, workspace file mode, live provider mode, and
  network mode.
- If a full arrow-key TUI dependency is not available, YonerAI uses a numbered
  menu fallback so the CLI remains standard-library and installable.

## Task progress and reviewer plan

Every `ask --auto` run can expose a task timeline:

- `classify`
- `route`
- `provider_select`
- `execute`
- `review`
- `result`

The visible reviewer/subagent plan is deterministic and public-safe. It can show
planner, researcher, reviewer, implementer, and tester roles, but it does not
launch uncontrolled autonomous agents.

## Provider and local LLM status

- Mock provider works by default and requires no key.
- Local LLM execution is loopback-only and explicit. `localhost`, `127.0.0.1`,
  and `::1` style endpoints are allowed; LAN hosts, tunnels, credentialed URLs,
  query strings, and fragments are rejected.
- OpenAI-compatible, Anthropic, and Gemini paths remain opt-in. Live external
  provider calls require explicit `--live` plus provider-specific environment
  opt-in.
- Provider keys are not printed, committed, or stored in the CLI config or run
  ledger.

## Safety model

- Network is off by default for the interactive runtime surface.
- Workspace file access remains the Workspace File Access Guard: only explicitly
  selected files inside an allowed workspace are read.
- Arbitrary local file access, folder crawling, PDF/image parsing, and real
  document summarization are not included.
- Shell/tools remain deny-by-default or dry-run only. SafeShell does not become
  arbitrary shell execution.
- Cloud-contract and Oracle behavior remain local-dev/stub visibility only.
  Private file content must not be routed to a cloud candidate.

## Install and packaging

This release validates the package entry point: after local editable install,
the generated `yonerai` command starts the CLI. The test uses a clean virtual
environment, seeds only the local build backend needed for editable packaging,
and installs `clients/cli` without using a package index for that entrypoint
check.

This is not a production network installer. It does not add npm, winget,
PowerShell remote download execution, PATH mutation, production signing keys, or
production trust stores.

The v0.5.0 release asset is represented by `releases/manifest.v0.5.0.json`.
Users can validate and dry-run-plan against it locally:

```powershell
yonerai manifest verify releases/manifest.v0.5.0.json --pretty
yonerai install plan --manifest releases/manifest.v0.5.0.json --pretty
yonerai update plan --manifest releases/manifest.v0.5.0.json --pretty
```

non-production placeholder signature status. It is not production install-ready
because production signature verification and production trust material are not
included in this public repository.

## What changed since v0.4.0-alpha.1

- Fixed post-release review debt in CLI task progress display and image context
  selection.
- Added Japanese-first Mission Control command coverage for tasks, Local LLM
  setup, live-provider mode, and network mode while keeping English aliases.
- Added provider capability negotiation output and subagent fallback visibility.
- Added a real install-like entrypoint test for the `yonerai` console script.
- Refreshed the AI CLI security/UX benchmark and carried concrete safeguards
  into the CLI surface.

## Known limitations

- This is stable only for the YonerAI CLI Local Runtime slice.
- It is not production Oracle, production control-plane, or Official Managed
  Cloud runtime.
- Official Managed Cloud remains external/contract-only in this public repo.
- Live Discord is not restored.
- External provider live calls are not default.
- Local LLM is loopback-only.
- SafeShell is not arbitrary shell execution.
- Workspace File Access Guard is not full file summarization.
- Persistent memory is not complete.
- Production installer, npm, and winget distribution are not ready.
- No Google login, production DB behavior, telemetry ingestion, deploy, or
  public tunnel is included.
- `src/cogs/ora.py` is not solved or broadly refactored.
