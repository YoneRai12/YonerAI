# 2026.05.26 — YonerAI v0.1.0-alpha.4 CLI Auto Runtime Slice

Release date: 2026-05-26
Tag: `v0.1.0-alpha.4`
Status: alpha pre-release
Previous pre-release: `v0.1.0-alpha.3`

## Summary

YonerAI v0.1.0-alpha.4 adds `yonerai ask --auto`, a public-safe CLI path that
classifies a task, chooses a local/stub execution route, returns a `run_id`, and
records redacted ledger events when a ledger is explicitly requested. This is a
developer alpha slice: it is not production Oracle, not Official Managed Cloud,
not live Discord, not arbitrary shell/file/tool execution, and not
production-ready.

## What you can actually run now

| Command | What it does | What it does not do | Mode |
| --- | --- | --- | --- |
| `yonerai ask "hello" --auto --json` | Classifies a simple public task as `instant`, routes it to `instant_local`, executes the mock provider, and returns a `run_id`. | Does not call an external provider, live Discord, production Oracle, or Official Managed Cloud. | Mock/local |
| `yonerai ask "hard public reasoning over public API docs" --auto --json` | Classifies a harder public task as `agent`, creates a deterministic reviewer/subtask plan, routes to `cloud_contract_candidate`, and runs the local-dev Oracle stub envelope with a `run_id`. | Does not use production Oracle or send raw private content/provider keys. | Local-dev Oracle stub |
| `yonerai ask "summarize this file" --auto --file <path> --workspace <dir> --provider anthropic --live --json` | Reads only the explicitly selected file inside the workspace allowlist, classifies it as local-file/private context, keeps it on the Hybrid/local route, and blocks the live external provider from receiving file content. | Does not crawl folders, parse PDFs/images, read outside the workspace, or send private file content to cloud-contract candidates. | Workspace File Access Guard |
| `yonerai ask "search the web for YonerAI alpha docs" --auto --json` | Uses the mock search adapter and records that live search was not performed. | Does not perform real web search or network calls. | Mock search |
| `yonerai ask "delete files and run shell command" --auto --json` | Classifies the task as dangerous and returns a deny/approval-required report. | Does not execute shell commands, mutate files, install packages, or run tools. | Deny-by-default |
| `yonerai ask "hard public reasoning over public API docs" --auto --ledger <local.jsonl> --json` | Writes redacted local-only run events such as `auto_runtime_decision`, `auto_reviewer_plan`, `oracle_stub_enqueued`, and `oracle_stub_result`. | Does not persist raw prompts, raw completions, provider keys, or arbitrary local paths. | Opt-in local ledger |
| `yonerai ask "hello" --auto --provider local --live --json` | Uses the loopback-only local LLM adapter when explicitly enabled and configured. | Does not allow non-loopback local LLM URLs and does not run by default. | Explicit local live |
| `yonerai demo --pretty` | Shows the public demo surface, including the `auto_runtime_ask` check. | Does not require credentials or live services. | Offline demo |
| `yonerai doctor --json` | Reports auto runtime availability, routes, defaults, and non-actions. | Does not run production health checks or live service calls. | Offline diagnostic |
| `yonerai start --guided --lang ja --pretty` | Shows first-run guidance that points users at `yonerai ask "hello" --auto --json`. | Does not install, deploy, mutate PATH, or send prompts to a hosted model. | Local first-run guidance |

## What changed since v0.1.0-alpha.3

- Added `yonerai ask --auto` as the main local-safe user path.
- Added task difficulty classes: `instant`, `task`, and `agent`.
- Added privacy classes: `public`, `private`, `local_file`, and `dangerous`.
- Added automatic route decisions: `instant_local`, `local_llm`,
  `hybrid_node`, `cloud_contract_candidate`, and `deny`.
- Connected mock provider execution, loopback-only local LLM execution, mock
  search, local-dev Hybrid/Oracle stub routing, reviewer/subtask planning, and
  redacted run ledger events behind one command.
- Hardened review findings so private/local-file auto routes cannot preserve or
  execute live external providers.
- Surfaced the auto runtime in `yonerai demo`, `yonerai doctor`, and
  `yonerai start --guided`.

## Real, local-dev, and stub boundaries

- Real: CLI classification, route decision, mock provider execution, local LLM
  loopback guard, workspace allowlist file read, local redacted ledger events,
  deny-by-default dangerous operation handling, and command output.
- Local-dev stub: Oracle queue request/result envelopes for public
  cloud-contract candidates.
- Contract-only: Official Managed Cloud and production Oracle remain outside
  this public repository.

## Security and boundary

- No production Oracle or Official Managed Cloud runtime is included.
- No production signing key or production trust store is included.
- No deploy, public tunnel, or live Discord connection is included.
- No provider keys are required for default tests or default demo paths.
- Live external provider calls remain opt-in only and are blocked for
  private/local-file auto routes.
- Local LLM endpoints are loopback-only.
- Private file content, provider keys, and raw prompt bodies are not sent to the
  Oracle stub/cloud-candidate path.
- Dangerous operations are denied; arbitrary shell/file/tool execution is not
  enabled.
- `src/cogs/ora.py` remains a legacy boundary and is not claimed solved.
- `reference_clawdbot` was not modified.

## Known limitations

- Not production-ready.
- Official Managed Cloud is not runnable from this public repository.
- Oracle/control-plane behavior is a local-dev stub/contract only.
- Live Discord is not restored.
- Persistent memory is not complete.
- Live web search is not default/live.
- SafeShell is not arbitrary shell.
- Workspace file access is an allowlist guard, not PDF/image parsing, folder
  crawling, arbitrary file access, or real LLM summarization.
- Production installer, npm, and winget are not ready.
- No Google login.
- No production DB behavior.
- No production telemetry ingestion.
