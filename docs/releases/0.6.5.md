Tag: `v0.6.5`

## Summary

v0.6.5 is a stable maintenance release for the YonerAI CLI Local Runtime
installer path. It keeps the short command `irm https://install.yonerai.com |
iex`, but makes broken or partial previous installs safer to recover from, adds
an explicit startup home header for `yonerai`, and keeps GitHub Releases as the
only installable-byte source.

## What you can run

| Command | What it does | What it does not do | Mode |
| --- | --- | --- | --- |
| `irm https://install.yonerai.com | iex` | Runs the Cloudflare static wrapper, downloads `install.ps1` and `install.ps1.sha256` from the latest stable GitHub Release, verifies the script hash, then launches the verified GitHub Release bootstrap. | Does not serve installer bytes from `yonerai.com`, use local PC files, mutate PATH by default, request admin, install services, store provider keys, or enable production cloud behavior. | Quick verified wrapper |
| `iex "& { $(irm https://github.com/YoneRai12/YonerAI/releases/latest/download/install.ps1) } -Execute -Launch"` | Runs the latest GitHub Release bootstrap directly, then verifies manifest, channel, artifact name, release ZIP SHA256, and ZIP size before install-like steps. | Does not verify the bootstrap script sidecar before execution. Use the `install.yonerai.com` command when you want the wrapper hash check first. | GitHub Release fallback |
| `& ([scriptblock]::Create((irm https://github.com/YoneRai12/YonerAI/releases/latest/download/install.ps1))) -Repair -Execute -Launch` | Moves an existing YonerAI install target aside to a timestamped backup, then installs from verified GitHub Release assets. | Does not delete the previous target. Does not use local/custom manifests or ZIP files. | Repair install |
| `& ([scriptblock]::Create((irm https://github.com/YoneRai12/YonerAI/releases/latest/download/install.ps1))) -Force -Execute -Launch` | Replaces an existing install target after preserving it as a timestamped backup. | Does not silently overwrite without the explicit `-Force` flag. | Forced reinstall with backup |
| `& ([scriptblock]::Create((irm https://github.com/YoneRai12/YonerAI/releases/latest/download/install.ps1))) -CleanRetry -Execute -Launch` | Treats a partial failed install as a retry case and backs it up before reinstalling. | Does not delete the partial folder in place. | Clean retry |
| `& ([scriptblock]::Create((irm https://github.com/YoneRai12/YonerAI/releases/latest/download/install.ps1))) -SetPath -Execute -Launch` | Installs and creates a user PATH command wrapper so future terminals can find `yonerai`. | Does not mutate PATH unless `-SetPath` is explicitly present. | Explicit PATH setup |
| `Get-Command yonerai -All` | Shows every `yonerai` command PowerShell can find so users can diagnose old installs shadowing the new one. | Does not change PATH or remove old files. | Local diagnosis |
| `yonerai update check --manifest manifest.v0.6.5.json --pretty` | Shows current version, manifest version, channel, artifact status, signature/trust state, and next safe command. | Does not download, install, mutate PATH, execute scripts, or contact an auto-updater. | Dry-run check |
| `yonerai update plan --manifest manifest.v0.6.5.json --pretty` | Shows the fuller dry-run update plan and all actions not performed. | Does not auto-update or apply changes. | Dry-run plan |

## Repair behavior

The installer now checks the install target before downloading the manifest or
ZIP. If the target already contains a previous runtime, a partial `source`
folder, or a partial `.venv`, the default run stops with a controlled message
and tells the user to choose one of these explicit paths:

- use `-Repair` to back up the existing target and install again
- use `-Force` when replacement is intentional
- use `-CleanRetry` for a failed or partial prior install
- use a different `-InstallDir`

The backup is a sibling folder with a timestamp suffix. The installer does not
delete the old target in place.

## Startup home

Running `yonerai` now starts with a YonerAI home header instead of only a dense
command dump. The header is cosmetic and terminal-safe:

- color is used only when terminal color is enabled
- `--color never` and non-TTY paths render plain text
- JSON outputs do not include ANSI/control sequences

## Source boundary

`yonerai.com/install` is command text and wrapper entry only. It must not serve
`install.ps1`, release manifests, ZIP artifacts, hash sidecars, or any local PC
file used for installation.

The install source of truth is GitHub Releases:

```text
https://github.com/YoneRai12/YonerAI/releases/latest/download/install.ps1
https://github.com/YoneRai12/YonerAI/releases/latest/download/install.ps1.sha256
https://github.com/YoneRai12/YonerAI/releases/download/v0.6.5/manifest.v0.6.5.json
https://github.com/YoneRai12/YonerAI/releases/download/v0.6.5/YonerAI-0.6.5.zip
```

## Security boundary

- Bootstrap execution can be hash-checked before running.
- The release ZIP SHA256 must match the manifest before extraction.
- Custom/local manifests and ZIP artifacts are rejected by `install.ps1`.
- Stable channel does not select alpha artifacts.
- Alpha still requires explicit `-Channel alpha`.
- No production signing key or production trust store is included.
- No PATH mutation is performed by default.
- User PATH setup is explicit through `-SetPath`.
- No service install, admin request, provider key storage, shared traffic,
  forced update, or auto-update apply is enabled.

## What changed since v0.6.4

- Added pre-download detection for existing, partial, or broken install targets.
- Added `-Repair`, `-Force`, and `-CleanRetry` recovery paths that preserve the
  old target as a timestamped backup.
- Added explicit `-SetPath` user PATH wrapper creation while keeping PATH
  mutation disabled by default.
- Added `Get-Command yonerai -All` guidance for diagnosing old command shadows.
- Added a YonerAI startup home header for the interactive CLI.
- Updated stable install documentation and site content to `v0.6.5`.

## Known limitations

- This is not full YonerAI cloud production.
- This is not a production-signed installer.
- Auto-update/apply is not implemented.
- npm, winget, Microsoft Store, and production installer channels are not ready.
- Production cloud runtime and live integrations are not included.
- Production account login is not included.
- External providers remain opt-in and keys are not printed or stored by the
  installer.
- Alpha memory, agent console, official API, and status API work remain
  prerelease/contract lanes and are not promoted into this stable release.

## Traceability
