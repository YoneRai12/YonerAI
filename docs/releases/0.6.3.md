# 2026.06.01 - YonerAI CLI Local Runtime v0.6.3 Verified Installer Hardening

Tag: `v0.6.3`

## Summary

v0.6.3 is a stable patch for the YonerAI CLI Local Runtime install path. It
adds a verified bootstrap command that checks `install.ps1.sha256` before script
execution, while keeping installable bytes on GitHub Releases and keeping
`yonerai.com/install` as plain command text only.

## Verified bootstrap flow

The verified install block performs these checks before execution:

1. Create a temporary bootstrap directory.
2. Download `install.ps1` from the latest stable GitHub Release asset redirect.
3. Download `install.ps1.sha256` from the same release asset redirect.
4. Reject a missing or malformed sidecar.
5. Compare the local `install.ps1` SHA256 with the sidecar.
6. Execute `install.ps1` only when the hash matches.

After that, `install.ps1` still verifies the manifest, selected channel,
versioned artifact name, ZIP SHA256, and ZIP size before install-like actions.

## Source boundary

`yonerai.com/install` is plain command text only. It must not serve
`install.ps1`, release manifests, ZIP artifacts, hash sidecars, or any local PC
file used for installation.

The install source of truth is GitHub Releases:

```text
https://github.com/YoneRai12/YonerAI/releases/latest/download/install.ps1
https://github.com/YoneRai12/YonerAI/releases/latest/download/install.ps1.sha256
https://github.com/YoneRai12/YonerAI/releases/download/v0.6.3/manifest.v0.6.3.json
https://github.com/YoneRai12/YonerAI/releases/download/v0.6.3/YonerAI-0.6.3.zip
```

## Security boundary

- Bootstrap execution can be hash-checked before running.
- The release ZIP SHA256 must match the manifest before extraction.
- Custom/local manifests and ZIP artifacts are rejected by `install.ps1`.
- Stable channel does not select alpha artifacts.
- Alpha still requires explicit `-Channel alpha`.
- No production signing key or production trust store is included.
- No PATH mutation, registry edit, service install, admin request, provider key
  storage, OpenAI shared traffic, production Oracle, or live Discord is enabled.

## What changed since v0.6.2

- Added a fail-closed verified bootstrap command to `yonerai.com/install`.
- Updated the stable bootstrap to target `v0.6.3`.
- Added the `v0.6.3` manifest and site release/press content.
- Made CLI update check/plan output include channel and artifact filename.
- Kept `install.ps1` plan-only by default and GitHub Release only.

## Known limitations

- This is not full YonerAI cloud production.
- This is not a production-signed installer.
- Auto-update/apply is not implemented.
- npm, winget, Microsoft Store, and production installer channels are not ready.
- Official Managed Cloud remains external/contract-only in this public repo.
- Production Oracle/control-plane runtime is not included.
- Google auth remains dry-run/contract only in the public repo.
- Live Discord is not restored.
- External providers remain opt-in and keys are not printed or stored by the
  installer.
