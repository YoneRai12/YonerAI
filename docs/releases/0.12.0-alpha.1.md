# 2026.06.01 - YonerAI v0.12.0-alpha.1 Memory Boundary Runtime

Release date: 2026-06-01
Tag: `v0.12.0-alpha.1`
Status: prerelease for local memory boundaries, sync preview contracts, and Japanese TUI memory/settings visibility
Previous release: `v0.11.0-alpha.1`

## Summary

YonerAI v0.12.0-alpha.1 adds the first public local memory boundary runtime. Users can add, inspect, forget, and preview sync decisions for local memory records while the public repo keeps cloud sync as a contract-only preview. The Japanese Mission Control CLI also gains a memory screen and a redesigned `/設定` flow that opens focused category screens instead of dumping every setting at once.

This release does not enable production cloud memory, production Google login, live Discord, OpenAI shared traffic, automatic local-to-cloud uploads, raw prompt ingestion for self-evolution, arbitrary shell/file/tool execution, production signing/trust, npm/winget, or a production network installer.

## Memory model

This release introduces public memory boundary records with:

- `MemoryRecord`
- `MemoryScope`: `session`, `local_private`, `cloud_account`, `shared_preference`, `project`, `procedural`, `self_evolution_signal`
- `MemorySource`: `user_explicit`, `conversation_summary`, `provider_result`, `sync_import`, `system_event`, `synthetic_signal`
- `MemorySensitivity`: `public`, `private`, `local_only`, `secret_like`
- `SyncPolicy`: `never_sync`, `cloud_to_local`, `local_to_cloud_requires_approval`, `shared_allowed`
- `MemoryStatus`: `active`, `archived`, `forgotten`, `pending_sync`, `rejected`

Every record has an id, timestamps, scope, sensitivity, sync policy, status, redacted summary, source reference, and audit reason. Secret-like text and local absolute paths are redacted before public output. Local path-shaped input is treated as local-only and never-sync.

## Local memory store

- The public store is local JSONL.
- Writes are explicit through `yonerai memory add`.
- Listing shows redacted summaries.
- Forget/delete operations are local only.
- Export is structured and redacted.
- The default store path is local config state, and CLI tests can pass an explicit `--store` or `--memory-store` path.

## Sync boundary

- Cloud-to-local sync can be previewed as a contract decision.
- Local-to-cloud sync is disabled by default.
- Local-to-cloud requires explicit approval and a safe sync policy.
- `local_private`, `local_only`, and `secret_like` memory cannot sync to cloud.
- Private file content, local node payloads, raw prompts, provider keys, secrets, and local absolute paths are excluded.
- Public fixtures do not contain raw private memory content.

## Ask and ledger integration

- `ask --auto` can use allowed local procedural or shared preference memory only when a memory store is explicitly provided.
- The run ledger records `memory_used` ids, not raw memory content.
- Ledger public output marks whether memory ids were persisted and confirms raw memory content was not persisted.

## TUI settings redesign

`/設定` now opens a category menu. Users can open focused screens such as:

- `/設定 言語`
- `/設定 提供元`
- `/設定 モデル`
- `/設定 安全`
- `/設定 記憶`
- `/設定 更新`
- `/設定 認証`
- `/設定 プライバシー`

Japanese command completion includes `/記憶`, `/メモリ`, and settings category candidates while English aliases remain compatible.

## Update policy

The update policy now separates normal, recommended, security, and critical update notices. The public CLI behavior is conservative:

- no forced silent update
- no auto-apply
- warn during active sessions instead of interrupting work
- show a follow-up prompt after a task when update notices are enabled
- show update screen first only for a configured critical update policy
- keep basic local mock chat available
- restrict only selected risky live/cloud/provider-sensitive features if a future critical policy requires a minimum safe version

## Self-evolution memory boundary

Public self-evolution signal memory is low-resolution and synthetic only. Allowed fields are feature id, surface, mode, outcome, dropoff stage, complaint class, provider class, and latency bucket. Raw prompts, PII, stable user tracking, local paths, and secret-like values are rejected.

## Security and boundary

- No production Oracle/cloud runtime.
- No Official Managed Cloud runtime in this public repository.
- No production Google login.
- No OpenAI shared traffic runtime.
- No live Discord or Discord token use.
- No deploy/public tunnel.
- No arbitrary shell/file/tool execution.
- No provider live call by default.
- No provider key output or config/ledger storage.
- Local LLM remains loopback-only.
- Local private memory is not uploaded automatically.
- Production signing keys and production trust stores are not included.
- Self-evolution remains proposal-only.
- `src/cogs/ora.py` is not solved by this release.
- `reference_clawdbot` is not touched.

## Known limitations

- This is a prerelease.
- It is not full YonerAI production.
- It is not production cloud memory.
- Production Oracle/control-plane runtime is not included.
- Official Managed Cloud runtime is not included in this public repository.
- Production Google login/passkey/account system is not enabled.
- Live Discord gateway restoration is not included.
- OpenAI shared traffic is not enabled.
- npm and winget channels are not available.
- Local private memory sync up is not automatic.
- SafeShell is not arbitrary shell execution.
- Workspace File Access Guard is not folder crawling, PDF/image parsing, arbitrary file access, or full file summarization.
- Persistent memory is still an alpha boundary runtime, not a complete product memory system.
- Self-evolution does not mutate code, GitHub, releases, deployments, or production configuration.
