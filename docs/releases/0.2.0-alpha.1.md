# 2026.05.26 — YonerAI v0.2.0-alpha.1 Real CLI Runtime Slice

Release date: 2026-05-26
Tag: `v0.2.0-alpha.1`
Status: alpha pre-release
Previous pre-release: `v0.1.0-alpha.4`

## Summary

YonerAI v0.2.0-alpha.1 turns the public CLI from a mostly release-demo surface
into a more usable local runtime shell. Users can start in Japanese, inspect
provider readiness, run `ask --auto` with visible routing and ledger state, use
mock/local-dev execution by default, and opt into loopback-only local LLM or
external provider paths only when explicitly configured. This is still an alpha
runtime slice, not production Oracle, Official Managed Cloud, live Discord, an
installer-ready distribution, or arbitrary shell/file execution.

## What changed since v0.1.0-alpha.4

- Added `yonerai providers` as the provider-readiness command.
- Improved `yonerai ask --auto --pretty --lang ja` with human route labels,
  provider decision, ledger state, and explicit non-actions.
- Improved `yonerai runs list/show --pretty --lang ja` so local ledger opt-in is
  understandable without reading JSON.
- Reframed mock workspace-file output as Workspace File Access Guard, not a real
  file-summary feature.
- Added a competitive AI CLI security/UX matrix and converted it into concrete
  implemented guardrails.
- Fixed release-gate security/runtime review debt: snake_case dangerous risk
  hints, self-host public reasoning routing, strict bool audit flags, real task
  route metadata for cloud-contract candidates, Oracle queue ledger env
  handling, and loopback relay pretty status.

## Provider runtime

- Mock provider is still the default and requires no key.
- Local LLM execution is real only when explicitly enabled with `--live` and a
  loopback-only URL.
- OpenAI-compatible, Anthropic, and Gemini paths are opt-in live paths. They
  require provider-specific env configuration and `--live`.
- Default tests and demos do not require provider keys and do not make live
  provider calls.
- Provider errors and setup status remain redacted.

## Auto routing

`yonerai ask --auto` classifies:

- difficulty: `instant`, `task`, `agent`
- privacy: `public`, `private`, `local_file`, `dangerous`
- route: `instant_local`, `local_llm`, `hybrid_node`,
  `cloud_contract_candidate`, or `deny`

Public simple tasks run through the local mock-safe path by default. Hard public
reasoning can use the local-dev Oracle stub envelope. Local-file/private work
stays local/hybrid and blocks external providers. Dangerous operations are
denied or marked approval-required; arbitrary shell/file/tool execution is not
enabled.

## Ledger and local files

The run ledger is opt-in and local-only. Use `--ledger <local.jsonl>` or
`YONERAI_RUN_LEDGER_PATH`. It stores run_id, status, classification,
route/provider decisions, and redacted event summaries. It does not persist raw
prompts, raw completions, provider keys, raw chain-of-thought, or arbitrary
local paths.

Workspace file behavior is a guard, not a document understanding feature. It
only accepts an explicitly selected UTF-8 text file inside the workspace
allowlist. PDF/image parsing, folder crawling, arbitrary file access, and real
LLM file summarization are not included.

## Security and boundary

- No production Oracle or Official Managed Cloud runtime is included.
- No production signing key or production trust store is included.
- No deploy, public tunnel, or live Discord connection is included.
- No provider keys are required for default tests or default demo paths.
- Live external provider calls remain opt-in only and are blocked for
  private/local-file auto routes.
- Local LLM endpoints are loopback-only.
- Private file content, provider keys, and raw prompt bodies are not sent to the
  Oracle stub/cloud-candidate path.
- Dangerous operations are denied; arbitrary shell/file/tool execution is not
  enabled.
- `src/cogs/ora.py` remains a legacy boundary and is not claimed solved.
- `reference_clawdbot` was not modified.

## Known limitations

- Not production-ready.
- Official Managed Cloud is not runnable from this public repository.
- Oracle/control-plane behavior is a local-dev stub/contract only.
- Live Discord is not restored.
- Persistent memory is not complete.
- Live web search is not default/live.
- SafeShell is not arbitrary shell.
- Workspace file access is an allowlist guard only.
- Production installer, npm, and winget are not ready.
- No Google login.
- No production DB behavior.
- No production telemetry ingestion.
