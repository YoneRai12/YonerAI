# YonerAI v0.1.0-alpha.1 Public Demo Slice

## Summary

YonerAI v0.1.0-alpha.1 is the first semantic pre-release candidate for the public runnable demo slice. It makes the clone-and-run developer experience visible through the YonerAI CLI without credentials, live Discord, Oracle access, provider API keys, production trust stores, or deployment.

This is a public demo milestone, not a production release.

## Quickstart

```powershell
python -m pip install -e .\clients\cli
yonerai demo --pretty
yonerai demo --json
yonerai quickstart
```

`yonerai quickstart` is an alias for the same deterministic public demo.

## YonerAI CLI

- `yonerai demo --pretty` prints the public demo in a readable release-check format.
- `yonerai demo --json` emits the stable `yonerai-public-demo/v1` contract with `schema_version: "1.0"`.
- `yonerai quickstart` runs the same demo path.
- The demo runs in-process and does not require a local Core API server.

## Demo Experience

The demo covers:

- `public_core`: local public health, mock message, and run contract.
- `mode_boundary`: Self-host public-local support, Hybrid Local Node contract/dev simulator support, and Managed Cloud as external contract-only.
- `route_preview`: safe public documentation tasks, private local file tasks, and dangerous shell tasks with boundary-aware routing.
- `hybrid_trust`: signed manifest verification, enrollment/session simulator, replay/tamper rejection, and dangerous-operation approval requirement.
- `managed_download`: managed URL acceptance and unsafe URL rejection.
- `self_evolution`: synthetic event, proposal-only output, scorecard, approval draft, and no auto issue/PR/merge/deploy.
- `limitations`: no production Oracle, live Discord, persistent memory, Google login, deploy, or official cloud runtime in this public repo.

## What Works

- Semantic pre-release version validation for `0.1.0-alpha.1`.
- GitHub release workflow classification for alpha/beta/rc pre-releases.
- A polished `yonerai demo` / `yonerai quickstart` CLI entry point.
- Managed download guard connection in the demo.
- Hybrid memory quarantine fixture connection in the demo, without persistence.
- Embed image URL SSRF guard for Discord vision handling, with regression tests.
- Public README, Japanese README, and CLI README quickstart instructions.

## Security / Boundary

- Official Managed Cloud remains `official_private_external_contract_only` in the public repository.
- The public repo contains compatibility contracts, client-facing stubs, and public-safe simulators only.
- Hybrid support is limited to Local Node contracts, signed envelope/session simulator behavior, and development fixtures.
- Self-evolution is synthetic and proposal-only.
- Embed image fetching now rejects loopback, private, link-local, metadata, reserved, non-HTTP, userinfo, unsafe redirect, and unsafe peer-IP targets.
