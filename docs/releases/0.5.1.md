# 2026.05.26 — YonerAI CLI Local Runtime v0.5.1 Distribution Trust Update

Release date: 2026-05-26
Tag: `v0.5.1`
Status: stable patch release for the local CLI runtime distribution path
Previous release: `v0.5.0`

## Summary

YonerAI CLI Local Runtime v0.5.1 is a stable patch release for distribution
trust hardening. It keeps the v0.5.0 interactive CLI behavior and adds the
missing release-gate pieces for the local distribution path: a plan-first
PowerShell bootstrap helper, a v0.5.1 release manifest, deterministic release
archive generation, yonerai.com install/release/press content, and clearer
source-available noncommercial licensing. This is still not a production
network installer and not full YonerAI cloud production.

## What users can run

| Command | What it does | What it does not do | Mode |
| --- | --- | --- | --- |
| `.\install-local.ps1` | Prints the local bootstrap plan from an extracted release folder or checkout. | Does not create `.venv`, install packages, mutate PATH, edit registry, install services, request admin, or execute remote scripts. | Plan-only |
| `.\install-local.ps1 -Execute -Launch` | Creates/reuses `.venv`, installs the local CLI package, then starts `yonerai`. | Does not run `irm ... \| iex`, does not install a production service, does not mutate PATH by default, and does not enable live providers. `pip` may fetch Python dependencies unless cached. | Explicit local install |
| `yonerai` | Starts the interactive Mission Control CLI when stdin is a TTY. | Does not start a GUI, deploy services, call live providers by default, or store provider keys. | Local CLI |
| `yonerai doctor --pretty --lang ja` | Explains local setup, provider readiness, manifest state, and safety boundaries in Japanese. | Does not install packages, mutate PATH, or connect to production services. | Offline diagnostic |
| `yonerai manifest verify releases/manifest.v0.5.1.json --pretty` | Validates the local v0.5.1 manifest contract, artifact name, SHA256 field, and placeholder signature status. | Does not download the release asset and does not treat placeholder trust as production signing. | Local verification |
| `yonerai install plan --manifest releases/manifest.v0.5.1.json --pretty` | Prints a dry-run install plan for the manifest artifact and explicit non-actions. | Does not download, install, mutate PATH, execute scripts, write registry, install services, or request admin. | Dry-run plan |
| `yonerai update plan --manifest releases/manifest.v0.5.1.json --pretty` | Compares the local version with the manifest version and shows whether an update would be needed. | Does not update files, download assets, execute remote code, or install packages. | Dry-run plan |

## Distribution trust changes

- `VERSION` and the CLI package metadata are set to `0.5.1`.
- `releases/manifest.v0.5.1.json` records the v0.5.1 GitHub Release asset name,
  URL, size, SHA256, and explicit non-production placeholder signature status.
- Release-specific manifests are excluded from generated source archives via
  `.gitattributes`, so a manifest can record the archive hash without changing
  the archive hash itself.
- `scripts/create_release.py` archives `HEAD^{tree}` with fixed entry mtimes
  instead of the commit object so the generated ZIP is content-tree based and
  reproducible across the release PR squash merge.
- `install-local.ps1` is documented as a local bootstrap helper, not a production
  network installer.

## Install and packaging

The release asset is expected to be:

- `YonerAI-0.5.1.zip`
- GitHub Release URL:
  `https://github.com/YoneRai12/YonerAI/releases/tag/v0.5.1`
- Manifest:
  `releases/manifest.v0.5.1.json`

Manual local install still uses a local virtual environment:

```powershell
cd "$HOME\Downloads\YonerAI-0.5.1"
python --version
.\install-local.ps1
.\install-local.ps1 -Execute -Launch
```

If PowerShell blocks local scripts, run the same helper without changing the
machine-wide execution policy:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\install-local.ps1 -Execute -Launch
```

## Known limitations

- This is stable only for the YonerAI CLI Local Runtime distribution path.
- It is not a production installer.
- No `irm ... | iex` install path is included.
- No network download-and-execute installer is included.
- No PATH mutation is performed by default.
- No production signing key, production trust store, key rotation service, or
  production release signing service is included.
- npm and winget distribution are not ready.
- Official Managed Cloud remains external/contract-only in this public repo.
- Production Oracle/control-plane runtime is not included.
- Live Discord is not restored.
- External provider live calls are not default.
- Local LLM remains loopback-only.
- SafeShell is not arbitrary shell execution.
- Workspace File Access Guard is not full file summarization.
- Persistent memory is not complete.
- No Google login, production DB behavior, telemetry ingestion, deploy, or
  public tunnel is included.
- `src/cogs/ora.py` is not solved or broadly refactored.

## Validation

Release gate validation for this release must include:

- `powershell -NoProfile -ExecutionPolicy Bypass -File .\install-local.ps1`
- `python scripts/create_release.py 0.5.1`
- SHA256 and size comparison between `YonerAI-0.5.1.zip` and
  `releases/manifest.v0.5.1.json`
- `yonerai manifest verify releases/manifest.v0.5.1.json --pretty`
- `yonerai install plan --manifest releases/manifest.v0.5.1.json --pretty`
- `yonerai update plan --manifest releases/manifest.v0.5.1.json --pretty`
- Targeted pytest, ruff, compileall, diff check, secret/local-path scan, and
  mojibake/hidden-Unicode scan.
