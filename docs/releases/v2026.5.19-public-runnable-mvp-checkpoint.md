## Open

YonerAI public development has moved from cleaned presentation into a verified local Core API smoke checkpoint.

This checkpoint means a new public checkout can install dependencies, run a credential-free Core API smoke test, start the local Core API, and receive a successful `/health` response.

It does not mean the full product, official cloud, live operations, Discord gateway, Web product, API product, CLI, native Japanese CLI, SNS lane, or self-host product is complete.

### Public Core API smoke

The public runnable MVP now has a credential-free Core API smoke path.

Verified commands:

```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
python -m pip install -U pip
pip install -r requirements.txt
$env:PYTHONPATH = "$PWD;$PWD\core\src"
$env:ORA_ALLOW_MISSING_SECRETS = "1"
python scripts/init_core_db.py
pytest tests/test_public_runnable_smoke.py tests/test_runtime_env_loader.py -q
pytest tests/test_distribution_node_mvp.py -q
python -m ora_core.main
Invoke-WebRequest -UseBasicParsing http://127.0.0.1:8001/health
```

Expected health body:

```json
{"ok": true}
```
