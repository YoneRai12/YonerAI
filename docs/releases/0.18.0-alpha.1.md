## Summary

YonerAI v0.18.0-alpha.1 completes the public CLI side of the staging Google
login bridge. The CLI can start a one-time staging auth bridge request, open or
show the browser URL, poll until the backend reports `linked`, fetch the minimal
staging account profile, and store only a public-safe YonerAI staging account
claim.

This is staging-only. The public CLI does not contain a Google client secret,
does not exchange Google OAuth tokens locally, does not store Google access
tokens, does not store refresh tokens, does not enable production Google login,
does not perform account sync, and does not upload local private content.

## What users can run

Set the staging origin first:

```powershell
$env:YONERAI_STAGING_AUTH_ORIGIN = "https://api-staging.yonerai.com"
```

| Command | What it does | What it does not do |
| --- | --- | --- |
| `yonerai auth google login --staging --bridge --open-browser --wait-linked --pretty --lang ja` | Starts the staging CLI bridge, opens the browser when possible, waits for linked state, and stores only a safe staging account claim. | Does not print or store Google tokens, refresh tokens, auth codes, provider keys, or local private content. |
| `yonerai auth status --pretty --lang ja` | Shows staging linked/not-linked state, redacted account label, production-login boundary, sync boundary, and privacy boundary. | Does not contact production Google login. |
| TUI `/認証` or `/auth` | Shows staging login availability and linked staging state inside the interactive CLI. | Does not enable account sync or shared traffic. |
| `yonerai auth google login --dry-run --pretty --lang ja` | Shows the local PKCE/state/loopback OAuth contract. | Does not open a browser or contact Google. |

Non-TTY users can omit `--open-browser`; the CLI prints a staging browser URL
instead. The URL is a one-time staging bridge URL, not a Google token.

## Linked-state behavior

- `POST /auth/cli/start` creates a one-time staging bridge request.
- The browser URL is opened only when `--open-browser` is explicit.
- `GET /auth/cli/poll/{request_id}` is polled only when `--wait-linked` or
  `--poll-request-id` is explicit.
- On linked, the public CLI sends the staging session claim only to
  `GET /v1/account/me`.
- The public CLI stores only a redacted YonerAI staging account claim under the
  local CLI config directory.
- Google access tokens, ID tokens, refresh tokens, auth codes, and provider keys
  are rejected if returned by the staging bridge.
- If linked state is not reached before timeout, the command fails closed with a
  controlled `staging_link_not_completed` error.

## Install / repair alpha

The stable one-command installer still installs the stable line by default:

```powershell
irm https://install.yonerai.com | iex
```

To install or repair this alpha explicitly from GitHub Release assets:

```powershell
& ([scriptblock]::Create((irm https://github.com/YoneRai12/YonerAI/releases/download/v0.18.0-alpha.1/install.ps1))) -Channel alpha -Version 0.18.0-alpha.1 -Repair -Execute -Launch
```

Add `-SetPath` only when you want the installer to create or refresh the user
PATH wrapper at `%LOCALAPPDATA%\YonerAI\bin\yonerai.cmd`.

The alpha repair command depends on the `v0.18.0-alpha.1` GitHub Release
assets. The release workflow uploads `install.ps1`, `install.ps1.sha256`,
`manifest.v0.18.0-alpha.1.json`, and `YonerAI-0.18.0-alpha.1.zip` together.

## Security and privacy boundary

- staging origins are allowlisted
- non-HTTPS public origins are rejected
- credentialed URLs and private IP origins are rejected
- bridge redirects are rejected before following them
- bridge paths containing backslashes or absolute URLs are rejected
- nested credential-like JSON keys are rejected
- Google client secrets are not required or supported in the public CLI
- token exchange is not performed in the public CLI
- token values are not printed
- refresh token plaintext storage is not enabled
- staging session token values are not stored
- local private memory, local file content, and local node content are not uploaded
- OpenAI shared traffic remains off
- production Google login remains unavailable

## Known limitations

- This is an alpha prerelease line, not the stable CLI Local Runtime line.
- Production Google login/passkey/account runtime is not included.
- Production AWS backend is not included.
- Production Oracle/cloud runtime is not included.
- Account sync is not performed by the public CLI.
- Live Discord is not restored.
- OpenAI shared traffic is not enabled.
- Local private memory is not uploaded automatically.
- Production cloud memory sync is not included.
- Production installer signing/trust store is not included.
- npm, winget, and auto-update apply are not included.
- Arbitrary shell/file/tool execution is not enabled.
- `src/cogs/ora.py` remains a legacy boundary and is not solved by this release.
- `reference_clawdbot` is not touched.
