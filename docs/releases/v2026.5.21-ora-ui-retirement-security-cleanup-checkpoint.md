# YonerAI v2026.5.21 ora-ui Retirement and Security Backlog Cleanup Checkpoint

Date: 2026-05-21
Status: public maintenance checkpoint
Scope: retire obsolete `ora-ui` surface and reduce dependency-security noise
Not a production release

## Open

YonerAI public development keeps the Core API and local LLM provider path as the active mainline.

The old `ora-ui` dashboard is no longer the product foundation and is removed from the active public surface instead of being dependency-remediated in place.

## Closed

- Retired the obsolete `ora-ui/` tree from the public repo.
- Removed old compose service entries that built the retired UI.
- Removed legacy launchers that still started the retired UI.
- Updated public docs to point new web smoke/demo work at `clients/web/`.
- Recorded the retirement decision in `docs/maintenance/ORA_UI_RETIREMENT_DECISION_2026_05_21.md`.

## Changed

- Dependabot triage now treats the former `ora-ui` alerts as expected to close after the default branch updates.
- Open PR backlog triage now identifies the old `ora-ui` Dependabot PRs as close-obsolete candidates after this cleanup reaches `main`.
- The current capability matrix says the retired UI is not a product foundation.

## Security and Hygiene

- Removing `ora-ui/package-lock.json` removes the manifest that accounted for 71 of 72 open Dependabot alerts before this cleanup.
- This checkpoint does not dismiss alerts manually.
- This checkpoint does not hide the remaining `clients/web` alert.
- No secret values, private repository dependency, provider API key, Discord token, deployment, or production release is added.

## Not Included

- production readiness
- final Web product UI
- Google login
- persistent memory
- Discord gateway completion
- external provider calls
- arbitrary remote local-provider URLs
- deployment
- `src/cogs/ora.py` implementation or rename
- `reference_clawdbot` repair

## Still Open

- non-`ora-ui` Dependabot PRs that need separate review
- old security PRs that need re-evaluation against current main
