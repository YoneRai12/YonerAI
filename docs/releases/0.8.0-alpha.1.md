# 2026.05.31 - YonerAI v0.8.0-alpha.1 Official Install/Auth Boundary

Release date: 2026-05-31
Tag: `v0.8.0-alpha.1`
Status: prerelease for the public-safe install/auth/privacy/self-evolution boundary
Previous release: `v0.7.0-alpha.1`

## Summary

YonerAI v0.8.0-alpha.1 makes the official-boundary alpha easier to inspect and
run from the CLI. It keeps the installer plan-first, Google OAuth dry-run-only,
OpenAI shared traffic off by default, and self-evolution proposal-only. It does
not enable production cloud, production Oracle, production Google login, live
Discord, production signing/trust, npm/winget, or a production network
installer.

## What changed since v0.7.0-alpha.1

- Added a v0.8 official install/auth/evolution boundary plan.
- Added an official self-evolution boundary contract.
- Added a private/official YonerAIOracle self-evolution handoff stub.
- Hardened `install.ps1` so it can read a local manifest in plan mode and
  report artifact name, SHA256 format, signature state, and production-trust
  absence without performing any install action.
- Cleaned the Japanese-first interactive CLI labels for auth/privacy/safety/
  self-evolution/update views while preserving English aliases.
- Added v0.8 yonerai.com release/press/install content foundations.

## Installer and distribution

`install.ps1` is still plan-only. It can display local manifest state, but it
does not perform a network install. The explicit local helper remains
`.\install-local.ps1 -Execute -Launch` from an extracted and verified release
folder.

The v0.8 manifest records a versioned ZIP artifact, SHA256, size, and
non-production/test trust status. Production signing keys, production trust
stores, a production signing service, npm, and winget are not included. The
manifest is the source of truth for the release asset hash because versioned
manifest files are excluded from generated source archives.

## Auth and shared traffic

- Google OAuth remains dry-run/contract only.
- OAuth PKCE, state, loopback redirect, and minimal scopes are visible.
- No production OAuth request, browser launch, token exchange, token printing,
  refresh-token plaintext storage, or account linking is performed.
- OpenAI shared traffic remains disabled by default.
- Private prompts, workspace-local file content, local memory records, local
  node payloads, provider keys, and secrets are excluded.

## Self-evolution

Public self-evolution remains proposal-only. It can show proposal queue state
and generate owner-reviewable proposal records from synthetic public-safe
fixtures. It cannot ingest production telemetry, raw prompts, raw completions,
support inbox contents, private runtime data, or stable user tracking. It
cannot open issues, create branches, open PRs, merge, tag, release, deploy, or
mutate production configuration.

## Security and boundary

- No production Oracle/cloud runtime.
- No Official Managed Cloud runtime in this public repository.
- No production Google login.
- No OpenAI shared traffic runtime.
- No live Discord or Discord token use.
- No deploy/public tunnel.
- No arbitrary shell/file/tool execution.
- No provider live call by default.
- No provider key output or config/ledger storage.
- Local LLM remains loopback-only.
- `src/cogs/ora.py` is not solved by this release.
- `reference_clawdbot` is not touched.

## Known limitations

- This is a prerelease, not a stable YonerAI cloud release.
- It is not full YonerAI production.
- It is not a production network installer.
- Production Oracle/control-plane runtime is not included.
- Official Managed Cloud runtime is not included in this public repository.
- Live Discord gateway restoration is not included.
- Google production login is not enabled.
- OpenAI shared traffic is not enabled.
- Production signing keys and production trust stores are not present.
- npm and winget channels are not available.
- SafeShell is not arbitrary shell execution.
- Workspace File Access Guard is not full file summarization.
- Persistent memory is not complete.
- Self-evolution is proposal-only and does not mutate code, GitHub, releases,
  deployments, or production configuration.
