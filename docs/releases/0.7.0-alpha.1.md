# 2026.05.31 - YonerAI v0.7.0-alpha.1 Official Bridge Foundation

Release date: 2026-05-31
Tag: `v0.7.0-alpha.1`
Status: prerelease for the public-safe official bridge foundation
Previous release: `v0.6.0`

## Summary

YonerAI v0.7.0-alpha.1 adds the first public-safe bridge layer between the
local CLI runtime and future official surfaces. It keeps v0.6.0 as the stable
CLI Local Runtime and adds proposal-only self-evolution, clearer site/release
content contracts, and bridge visibility in the interactive CLI. It does not
turn on production cloud, production Google login, OpenAI shared traffic, live
Discord, production Oracle, or a production network installer.

## What changed since v0.6.0

- Added a public-safe self-evolution proposal queue foundation.
- Added `yonerai evolve status`, `yonerai evolve simulate`, and
  `yonerai evolve proposals list/show`.
- Added `/自己進化` and `/evolve` to the interactive CLI.
- Added synthetic low-resolution queue fixtures and validation that rejects raw
  prompt/completion fields, PII-like fields, local paths, live URLs,
  secret-shaped values, and GitHub/deploy/release mutation fields.
- Added v0.7 prerelease site/release/press content contracts for the future
  `yonerai.com` surface while keeping GitHub Release as the distribution source
  of truth.
- Kept Google OAuth as dry-run/contract only and OpenAI shared traffic off by
  default.

## Self-evolution proposal queue

The queue accepts only synthetic or local public-safe low-resolution signals:

- `feature_id`
- `surface`
- `mode`
- `outcome`
- `dropoff_stage`
- `complaint_class`
- `provider_class`
- `latency_bucket`

Each generated proposal includes:

- user impact
- frequency hint
- privacy risk
- implementation cost
- provider-independence impact
- same-experience impact
- test plan
- rollback plan
- release-note draft
- X/social-post draft

Approval states are `proposed`, `approved`, `rejected`, and `needs_owner`.
This prerelease does not implement automatic proposal approval or automatic
repo/GitHub/deploy/release mutation.

## Auth and shared traffic

- Google OAuth remains dry-run/contract only.
- OAuth PKCE, state, loopback redirect, and minimal scopes are documented and
  visible.
- No production OAuth request, browser launch, token exchange, token printing,
  refresh-token plaintext storage, or account linking is performed.
- OpenAI shared traffic remains disabled by default.
- Private prompts, workspace-local file content, local memory records, local
  node payloads, provider keys, and secrets are excluded from shared-traffic
  policy.

## Installer and distribution

- v0.6.0 remains the stable CLI Local Runtime release.
- v0.7.0-alpha.1 is a prerelease bridge foundation.
- `install-local.ps1` remains explicit local bootstrap, not a production
  network installer.
- `install.ps1` remains a dry-run skeleton for a future installer lane.
- The prerelease manifest records a versioned ZIP artifact, SHA256, size, and
  placeholder non-production signature status.
- Production signing keys, production trust stores, npm, winget, and remote
  execute install flows are not included.

## Security and boundary

- No production Oracle/cloud runtime.
- No Official Managed Cloud runtime in this public repository.
- No production Google login.
- No OpenAI shared traffic runtime.
- No live Discord or Discord token use.
- No deploy/public tunnel.
- No arbitrary shell/file/tool execution.
- No provider live call by default.
- No provider key output or config/ledger storage.
- Local LLM remains loopback-only.
- `src/cogs/ora.py` is not solved by this release.
- `reference_clawdbot` is not touched.

## Known limitations

- This is a prerelease, not the stable CLI Local Runtime release.
- It is not full YonerAI cloud production.
- It is not a production network installer.
- Production Oracle/control-plane runtime is not included.
- Official Managed Cloud runtime is not included in this public repository.
- Live Discord gateway restoration is not included.
- Google production login is not enabled.
- OpenAI shared traffic is not enabled.
- Production signing keys and production trust stores are not present.
- npm and winget channels are not available.
- SafeShell is not arbitrary shell execution.
- Workspace File Access Guard is not full file summarization.
- Persistent memory is not complete.
- Self-evolution is proposal-only and does not mutate code, GitHub, releases,
  deployments, or production configuration.
