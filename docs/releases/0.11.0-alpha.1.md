# 2026.06.01 - YonerAI v0.11.0-alpha.1 Account Sync and Oracle API Foundation

Release date: 2026-06-01
Tag: `v0.11.0-alpha.1`
Status: prerelease for public account sync contracts, Official API fixtures, and CLI/TUI sync visibility
Previous release: `v0.10.0-alpha.1`

## Summary

YonerAI v0.11.0-alpha.1 adds the public side of the future official account
sync and Oracle API boundary. Users can inspect Google auth dry-run state,
privacy state, and a new cloud/local sync boundary from the CLI and Japanese
Mission Control TUI. The public repository now includes contract objects,
fixtures, conformance tests, and a sanitized private YonerAIOracle handoff so
public CLI Local Runtime and future official backend work can stay aligned.

This is not production Official Managed Cloud. It does not enable production
Google login, production Oracle, live Discord, OpenAI shared traffic runtime,
automatic local-to-cloud uploads, production signing/trust, npm/winget, or a
production network installer.

## What changed since v0.10.0-alpha.1

- Added `ora_core.official` public contract helpers for account identity,
  Google auth session, local user profile, cloud account link state, sync
  refs/envelopes/decisions/audit, official API fixtures, and rate-limit policy.
- Added `yonerai sync status`, `yonerai sync preview`,
  `yonerai sync approve --dry-run`, `yonerai sync api-contract`, and
  `yonerai sync rate-limit`.
- Added `/同期` and `/sync` to the interactive Mission Control CLI.
- Added Official API Contract 0.1 docs plus JSON schema and fixture.
- Added sanitized private YonerAIOracle self-evolution handoff for future
  private backend alignment.
- Hardened sync preview semantics so blocked previews return `ok=false`, and
  pretty output distinguishes `ok`, `warn`, and `fail` states.

## Account and sync boundary

- Google auth remains dry-run/contract only in the public repo.
- OAuth contract requires PKCE, state, loopback redirect, and minimal
  `openid email profile` scopes.
- Cloud conversation to local sync is allowed only after a linked account and a
  user-selected cloud conversation.
- Local private conversation to cloud is disabled by default.
- Local to cloud sync requires explicit approval and audit reason.
- Private file content, local memory, local node payloads, raw prompts,
  provider keys, secrets, and local absolute paths are excluded by default.

## Official API contract

The public fixture names these future official endpoints:

- `GET /v1/account/me`
- `GET /v1/conversations`
- `POST /v1/sync/preview`
- `POST /v1/sync/approve`
- `POST /v1/oracle/runs`
- `GET /v1/oracle/runs/{id}`
- `GET /v1/rate-limit`
- `GET /v1/status`

These endpoints are contract/fixture only in this public repository. Production
backend implementation belongs to the private/official YonerAIOracle lane.

## Rate limit and self-evolution boundary

- Rate-limit policy is split into user quota, device quota, provider quota,
  shared/free traffic policy, abuse prevention, and local fallback.
- OpenAI shared traffic remains disabled by default.
- The public repo does not claim free usage or owner/org eligibility.
- Public self-evolution remains proposal-only with synthetic low-resolution
  signals.
- The public repo does not ingest raw prompts, PII, real telemetry, or support
  transcripts.
- The public repo does not auto-create PRs, merge, release, deploy, or mutate
  production configuration.

## Private YonerAIOracle status

A separate private worker lane prepared a sanitized backend skeleton patch in an
isolated private worktree. Public release notes intentionally do not include
private paths, production routes, secrets, hostnames, runtime inventory, or
control-plane internals. The private patch was not committed to this public
repository.
