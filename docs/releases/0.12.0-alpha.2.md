# 2026.06.01 - YonerAI v0.12.0-alpha.2 Memory UX and Update Notice

Release date: 2026-06-01
Tag: `v0.12.0-alpha.2`
Status: prerelease for memory UX, update notice behavior, and distribution trust-root hardening
Previous release: `v0.12.0-alpha.1`

## Summary

YonerAI v0.12.0-alpha.2 publishes the memory UX work that was recovered from the local branch and merged through PR #488. It makes the local memory boundary easier to use from the Japanese-first interactive CLI, keeps status/settings metadata-only, blocks memory writes when memory is disabled, and makes update notices respect settings during the current session. It also includes the security and installer trust-root fixes that landed after v0.12.0-alpha.1.

This release does not enable production cloud memory, production Google login, live Discord, OpenAI shared traffic, automatic local-to-cloud uploads, raw prompt ingestion for self-evolution, arbitrary shell/file/tool execution, production signing/trust, npm/winget, or a production network installer.

## Memory UX changes

- `/記憶`, `/メモリ`, and `/memory` now expose memory status, add, list, forget, and sync preview actions in the interactive CLI.
- `/設定 記憶` is a focused memory settings category instead of a giant settings dump.
- Memory settings include enabled state, default scope, cloud-to-local preview, local-to-cloud approval boundary, and self-evolution signal memory status.
- When memory is disabled, explicit TUI memory writes are blocked with a controlled message.
- Memory status and settings stay metadata-only. They show counts and boundary flags, not memory text.
- Explicit list commands remain the place where redacted memory summaries are shown.
- `ask --auto` and TUI responses show `memory_used` ids only; raw memory content is not placed in the ledger.

## Sync boundary

- Cloud-to-local sync remains a preview contract only.
- Cloud-to-local preview can be disabled through local config and is honored by both TUI and standalone CLI paths.
- Local-to-cloud sync is disabled by default.
- Local-to-cloud requires explicit approval and a safe sync policy in future private/official work.
- `local_private`, `local_only`, and `secret_like` memory cannot sync to cloud.
- Private file content, local node payloads, raw prompts, provider keys, secrets, and local absolute paths are excluded.

## Update notice behavior

- Startup can show a non-blocking update notice when configured.
- Post-task update notices reuse the startup check instead of running update checks after every task.
- Turning update notices off during a TUI session takes effect immediately.
- No silent update or auto-apply is implemented.
- Basic local mock chat remains available even when a future critical update policy is displayed.

## Security and distribution hardening

- Local memory redaction now covers additional Unix-style local roots such as workspace, opt, srv, mnt, media, volumes, private, usr, and app paths while avoiding slash-command/API false positives.
- Security scan allowlists were adjusted for current main without weakening provider-key handling.
- Self-evolution proposal pretty output was hardened without enabling raw prompt ingestion.
- The installer short wrapper trust root was separated from GitHub latest lookup: the wrapper may discover latest metadata, but execution is restricted to trusted release tags and pinned script hashes.
- The public repo still does not host production signing keys or production trust stores.

## Known limitations

- This is a prerelease.
- It is not full YonerAI production.
- Production cloud memory is not implemented.
- Production Oracle/control-plane runtime is not included.
- Official Managed Cloud runtime is not included in this public repository.
- Production Google login/passkey/account system is not enabled.
- Live Discord gateway restoration is not included.
- OpenAI shared traffic is not enabled.
- npm and winget channels are not available.
- Local private memory sync up is not automatic.
- SafeShell is not arbitrary shell execution.
- Workspace File Access Guard is not folder crawling, PDF/image parsing, arbitrary file access, or full file summarization.
- Persistent memory remains an alpha boundary runtime, not a complete product memory system.
- Self-evolution does not mutate code, GitHub, releases, deployments, or production configuration.
- `src/cogs/ora.py` is not solved by this release.
- `reference_clawdbot` is not touched.
