# 2026.06.01 - YonerAI CLI Local Runtime v0.6.1 One-Command Install

Tag: `v0.6.1`

## Summary

v0.6.1 is a stable patch for the YonerAI CLI Local Runtime distribution path.
It makes the Windows install path usable as one PowerShell command while keeping
the local-PC-hosted `yonerai.com` surface out of the installer trust chain.

## What you can run

| Command | What it does | What it does not do | Mode |
| --- | --- | --- | --- |
| `& ([scriptblock]::Create((irm https://github.com/YoneRai12/YonerAI/releases/latest/download/install.ps1))) -Execute -Launch` | Downloads the latest stable GitHub Release bootstrap, verifies the release ZIP hash through the manifest, installs the local CLI runtime, and launches `yonerai`. | Does not fetch installer bytes from `yonerai.com`, mutate PATH by default, request admin, install services, store provider keys, or enable production cloud. | GitHub Release bootstrap |
| `$b="https://github.com/YoneRai12/YonerAI/releases/latest/download"; ... install.ps1.sha256 ...` | Hash-checks the bootstrap script before running it. | Does not use a local PC-hosted script source. | Verified bootstrap |
| `yonerai install status --pretty` | Shows the install source policy, selected stable channel, selected artifact, signature/trust warning, and recommended command. | Does not download or install. | Local status |
| `yonerai update check --channel stable --pretty` | Checks the current local `VERSION` against the stable release manifest in the checkout. | Does not auto-download or auto-install updates. | Dry-run check |
| `yonerai update check --channel alpha --pretty` | Checks the current local `VERSION` against the alpha release manifest in the checkout. | Does not make alpha the default install path. | Explicit alpha check |
| `yonerai` | Starts the local interactive CLI. | Does not enable live providers, production Oracle, Official Managed Cloud, Google login, or live Discord. | Local CLI |

## Distribution source

`yonerai.com/install` is a command page only. It must not serve `install.ps1`,
release manifests, ZIP artifacts, or any local PC file used for installation.
Installable bytes come from GitHub Release assets only.

The recommended command uses:

```text
https://github.com/YoneRai12/YonerAI/releases/latest/download/install.ps1
```

That URL resolves to the latest stable GitHub Release asset. The downloaded
bootstrap then downloads versioned assets from the same public GitHub Release
line, such as `manifest.v0.6.1.json` and `YonerAI-0.6.1.zip`.

## Security boundary

- `yonerai.com` is not the script, manifest, or ZIP host.
- Custom/local manifests and ZIP artifacts are rejected by `install.ps1`.
- The release ZIP SHA256 must match the manifest before extraction.
- `install-local.ps1` runs only after it is found inside the verified GitHub
  Release archive.
- No production signing key or production trust store is included.
- No PATH mutation, registry edit, service install, admin request, provider key
  storage, OpenAI shared traffic, production Oracle, or live Discord is enabled.

## What changed since v0.6.0

- Added a GitHub Release bootstrap path for one-command Windows install.
- Added `install.ps1` and `install.ps1.sha256` to the release workflow assets.
- Added `yonerai install status` so users can inspect the install trust source.
- Updated install/update defaults to select the stable channel by default while
  keeping alpha explicit.
- Rewrote `yonerai.com/install` as command-only content that points to GitHub
  Release assets and forbids local PC-hosted installer files.
- Updated manifest schema/validation for the GitHub Release bootstrap method.

## Known limitations

- This is not full YonerAI cloud production.
- This is not a production-signed installer.
- npm, winget, Microsoft Store, and production installer channels are not ready.
- Official Managed Cloud remains external/contract-only in this public repo.
- Production Oracle/control-plane runtime is not included.
- Google auth remains dry-run/contract only in the public repo.
- Live Discord is not restored.
- External providers remain opt-in and keys are not printed or stored by the
  installer.
