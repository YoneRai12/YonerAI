# YonerAI v2026.5.20 Public Core Message MVP Checkpoint

Date: 2026-05-20  
Status: public core message MVP checkpoint  
Scope: local Core API message contract / offline mock responder / refactor Step 0.1 hardening  
Not a production release

## Open

YonerAI public development has moved from a health-only smoke checkpoint to the first credential-free public message contract.

This checkpoint still describes a local public MVP. It is not a Web chat product, not provider generation, not Discord gateway completion, and not production readiness.

## Closed

- Added a public Core API message smoke endpoint:
  - `POST /v1/public/messages`
  - `mode: "mock"` or `mode: "offline"`
  - deterministic offline mock reply
  - no provider API key required
  - no Discord token required
  - no private repository required
- Kept the existing `/v1/messages` run-submission path separate from the public mock contract.
- Added public-safe tests for health plus mock/offline message behavior.
- Hardened the `ora.py` static import map follow-up:
  - analyzer exclusions are constants
  - import-map tests capture runtime-module baseline before loading the analyzer

## Changed

- README and README_JP now describe the current public MVP as health smoke plus mock/offline message contract.
- The capability matrix now distinguishes the mock/offline message endpoint from Web UI chat, memory, Discord gateway completion, and provider live generation.
- Release notes index now lists this checkpoint as the latest public progress note.

## Security and hygiene

- The public message endpoint does not call OpenAI, Gemini, Anthropic, Discord, SNS, web search, private repositories, or any external provider.
- The endpoint does not persist memory or conversation history.
- The endpoint rejects unsupported modes and obvious secret-like or private-path payload markers.
- This checkpoint does not rewrite git history and does not create a GitHub Release tag.
- No secret values should be included in changed files.
