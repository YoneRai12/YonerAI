# YonerAI v2026.5.20 Local LLM Conversation MVP Checkpoint

Date: 2026-05-20
Status: public Local LLM conversation MVP checkpoint
Scope: loopback-only local model adapter / public Core message contract / dependency and PR backlog gate
Not a production release

## Open

YonerAI public development has moved from mock/offline message smoke to the first local conversation path.

This checkpoint is still local and public-safe. It is not production readiness, not official cloud, not Google login, not persistent memory, not Discord gateway completion, and not ChatGPT-equivalent feature parity.

## Closed

- Added loopback-only local LLM mode to `POST /v1/public/messages`.
- Added an Ollama-compatible local adapter for `/api/chat`.
- Preserved existing `mock` and `offline` modes.
- Added tests for:
  - loopback URL acceptance
  - remote/LAN/credential URL rejection
  - fake local LLM response parsing
  - safe timeout/error behavior
  - endpoint local mode with fake adapter
  - non-loopback client rejection for local mode
- Added an optional local smoke script for users who already run a local Ollama-compatible runtime.
- Re-triaged Dependabot alerts and open PR backlog before building on the next lane.

## Changed

- README, README_JP, and the capability matrix now describe local LLM conversation as optional and loopback-only.
- The v7.7 implementation addendum records that this is still not a v7.8 replacement.
- The old `ora-ui` surface is explicitly kept out of this product foundation.

## Security and Hygiene

- Local LLM mode only accepts loopback endpoints such as `localhost`, `127.0.0.1`, or `::1`.
- Arbitrary remote provider URLs, LAN hosts, file URLs, embedded credentials, query strings, and fragments are rejected.
- Non-loopback clients cannot use local mode.
- No OpenAI, Anthropic, Gemini, Discord, SNS, web search, or external provider call is added.
- No provider API key, Discord token, Google login, private repository, deployment, or release tag is required.
- Remaining Dependabot critical/high alerts are concentrated in deferred `ora-ui`, not the Core local LLM path.
