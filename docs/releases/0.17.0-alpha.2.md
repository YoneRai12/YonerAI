## Summary

YonerAI v0.17.0-alpha.2 repairs the staging Google login alpha path. The public
CLI can now explicitly call the staging CLI bridge, start a one-time auth
request, and poll the staging request state without enabling production Google
login.

This is staging-only. The public CLI does not contain a Google client secret,
does not exchange OAuth tokens, does not store refresh tokens, does not perform
account sync, and does not upload local private content.

## What users can run

| Command | What it does | What it does not do |
| --- | --- | --- |
| `yonerai auth status --pretty --lang ja` | Shows auth, staging, sync, shared-traffic, and privacy state. | Does not log in or contact production auth. |
| `yonerai auth google login --dry-run --pretty --lang ja` | Shows the PKCE/state/loopback Google OAuth contract. | Does not open a browser, exchange tokens, or store refresh tokens. |
| `yonerai auth google login --staging --pretty --lang ja` | Shows the staging login contract and staging auth URL when configured. | Does not call the CLI bridge unless `--bridge` is added. |
| `yonerai auth google login --staging --bridge --pretty --lang ja` | Calls the staging CLI bridge start endpoint and shows the safe next auth URL. | Does not print tokens, store refresh tokens, or enable production login. |
| `yonerai auth google login --staging --poll-request-id <id> --pretty --lang ja` | Polls a one-time staging bridge request id. | Does not print staging session token values. |

Set the staging origin first:

```powershell
$env:YONERAI_STAGING_AUTH_ORIGIN = "https://api-staging.yonerai.com"
yonerai auth google login --staging --bridge --pretty --lang ja
```

Add `-SetPath` only when you want the installer to create or refresh the user
PATH wrapper at `%LOCALAPPDATA%\YonerAI\bin\yonerai.cmd`.

## Security and privacy boundary

The bridge path is fail-closed:

- network calls are off unless `--bridge` or `--poll-request-id` is explicit
- staging origins are allowlisted
- non-HTTPS public origins are rejected
- credentialed URLs and private IP origins are rejected
- bridge redirects are rejected before following them
- non-JSON HTTP errors are controlled and do not leak raw endpoints
- bridge paths containing backslashes or absolute URLs are rejected
- nested credential-like JSON keys are rejected
- Google client secrets are not required or supported
- token exchange is not performed in the public CLI
- refresh token plaintext storage is not enabled
- local private memory, local file content, and local node content are not uploaded
- OpenAI shared traffic remains off

## Known limitations

- This is an alpha prerelease line, not the stable CLI Local Runtime line.
- Production Google login/passkey/account runtime is not included.
- Production AWS backend is not included.
- Production Oracle/cloud runtime is not included.
- Account sync is not performed by the public CLI.
- Live Discord is not restored.
- OpenAI shared traffic is not enabled.
- Local private memory is not uploaded automatically.
- Production cloud memory sync is not included.
- Production installer signing/trust store is not included.
- npm, winget, and auto-update apply are not included.
- Arbitrary shell/file/tool execution is not enabled.
- `src/cogs/ora.py` remains a legacy boundary and is not solved by this release.
- `reference_clawdbot` is not touched.
