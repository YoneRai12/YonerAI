# v2026.5.20.2 Conversation Session Scaffold Checkpoint

Status: public checkpoint note. This is not a production release and does not create a GitHub Release tag.

## Open

- Conversation Session Scaffold is now part of the public Core API contract surface for review.
- Feature inventory and releaseability map now describe scattered public-repo functionality before promoting more features.

## Closed

- Public `POST /v1/public/messages` can return `session_id`, `turn_index`, `history_count`, and `memory_persisted`.
- The scaffold groups messages by `session_id` / `conversation_id` inside the running Core API process.
- Temporary `clients/web` passes the returned session id into follow-up public message requests.

## Changed

- Added an in-memory public session metadata store.
- Added unique generated session ids for omitted `session_id` values and a bounded in-memory store.
- Updated public message schemas and tests for session metadata.
- Added public-safe feature inventory and releaseability map docs.
- Updated public capability wording to distinguish session metadata from memory.

## Security and hygiene

- No provider API key is required.
- No Discord token is required.
- No external provider call is added.
- No arbitrary remote provider URL is allowed.
- No persistent message memory is added.
- Session metadata stores counts and identifiers only; message content is not persisted by this scaffold.
- The in-memory store has a maximum entry count and evicts the oldest session metadata when full.
