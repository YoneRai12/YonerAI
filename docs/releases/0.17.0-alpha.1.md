## Summary

YonerAI v0.17.0-alpha.1 adds public CLI/TUI support for the staging Google
login contract. Users can now see whether staging auth is available, generate a
public-safe staging authorization URL, and choose local-only or auth-check
onboarding without enabling production Google login.

This is staging/auth-contract UX only. The public CLI does not contain a Google
client secret, does not exchange OAuth tokens, does not store refresh tokens,
does not perform account sync, and does not upload local private content.

## What users can run

| Command | What it does | What it does not do | Mode |
| --- | --- | --- | --- |
| `yonerai auth status --pretty --lang ja` | Shows Google auth, staging availability, account sync, and privacy state in Japanese. | Does not perform login or contact production Google OAuth. | Local status |
| `yonerai auth google login --dry-run --pretty --lang ja` | Shows the installed-app OAuth PKCE/loopback/state contract. | Does not open a browser, contact Google, print tokens, or store refresh tokens. | Dry-run contract |
| `yonerai auth google login --staging --pretty --lang ja` | Generates a staging YonerAI auth URL when `YONERAI_STAGING_AUTH_ORIGIN` or `YONERAI_OFFICIAL_API_STAGING_ORIGIN` points to an allowlisted staging host. | Does not exchange tokens in the public CLI or enable production login. | Staging contract |
| TUI `/認証` or `/auth` | Shows local-only, dry-run, staging, sync, shared-traffic, and private-upload status. | Does not enable account sync or shared traffic. | Interactive auth screen |
| First launch account step | Lets the user choose local-only, Google login check, or later. | Does not block local CLI use. | Interactive onboarding |

## Staging origin configuration

Set one of these environment variables before using the staging command:

```powershell
$env:YONERAI_STAGING_AUTH_ORIGIN = "https://api-staging.yonerai.com"
yonerai auth google login --staging --pretty --lang ja
```

Equivalent staging-origin environment variable:

```powershell
$env:YONERAI_OFFICIAL_API_STAGING_ORIGIN = "https://api-staging.yonerai.com"
yonerai auth google login --staging --pretty --lang ja
```

Allowed public staging hosts:

- `api-staging.yonerai.com`
- `staging.yonerai.com`

Localhost development origins are rejected by default. They are accepted only
when `YONERAI_STAGING_AUTH_ALLOW_LOCALHOST_DEV=1` is explicitly set for local
development.

## Security and privacy boundary

The staging login path is fail-closed:

- non-HTTPS public staging origins are rejected
- credentialed URLs are rejected
- private IPs and non-allowlisted hosts are rejected
- malformed origin and redirect ports return controlled errors
- unexpected redirect hosts are rejected
- client secrets are not required or supported in the public CLI
- tokens are not printed
- token exchange is not performed in the public CLI
- refresh token plaintext storage is not enabled
- local private memory, local file content, and local node content are not
  uploaded
- OpenAI shared traffic remains off

## Sync boundary

Account sync remains a boundary contract in this prerelease:

- cloud-to-local sync is future/staging only unless a backend-linked account
  exists
- local-to-cloud sync is disabled by default
- local-to-cloud sync requires explicit approval
- local private memory, private file content, and local node content are
  excluded by default
- every sync preview must carry an audit reason

## Known limitations

- This is an alpha prerelease line, not the stable CLI Local Runtime line.
- Production Google login/passkey/account runtime is not included.
- Production AWS backend is not included.
- Production Oracle/cloud runtime is not included.
- Account sync is not performed by the public CLI.
- Live Discord is not restored.
- OpenAI shared traffic is not enabled.
- Local private memory is not uploaded automatically.
- Production cloud memory sync is not included.
- Production installer signing/trust store is not included.
- npm, winget, and auto-update apply are not included.
- Arbitrary shell/file/tool execution is not enabled.
- `src/cogs/ora.py` remains a legacy boundary and is not solved by this release.
- `reference_clawdbot` is not touched.
