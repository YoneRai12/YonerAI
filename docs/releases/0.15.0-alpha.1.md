## Summary

YonerAI v0.15.0-alpha.1 adds a public-safe Status API bridge for CLI, TUI,
doctor, status.yonerai.com alignment, and future private/AWS backend work. The
public repository contains contracts, schemas, fixtures, local readers, and
tests only. It does not add production AWS, production Oracle/cloud runtime,
production Google login, live Discord, provider keys, or private runtime
inventory.

This release also fixes the release blocker found after PR #500: sourced status
feeds now reject private/reserved IP URLs, internal hostnames, AWS ARNs, local
paths, and secret-like markers before public JSON is printed.

## Status API boundary

The public repo can define and read the public status shape. It cannot prove or
operate the private backend.

Public repo state:

- contract and schema files
- public fixtures
- local CLI/TUI readers
- doctor summary integration
- conformance tests
- private/AWS handoff document

Private or official lane required later:

- real AWS/API Gateway/CloudFront/WAF deployment
- real status aggregation from private monitoring
- real incident workflow
- production rate-limit enforcement
- production secrets management
- production observability
- status.yonerai.com production publishing pipeline

## Private endpoint rejection

When a local file or allowlisted network status feed is provided, YonerAI now
checks public-facing status payload fields before printing JSON.

Rejected examples include:

- private IPv4 ranges such as `10.x.x.x`, `172.16.x.x` through `172.31.x.x`,
  and `192.168.x.x`
- loopback, link-local, metadata-service, multicast, reserved, or unspecified IP
  addresses
- IPv6 loopback, unique-local, link-local, multicast, reserved, or unspecified
  addresses
- raw URLs pointing at private or internal hosts
- internal hostname suffixes such as `.internal`, `.local`, `.lan`, `.corp`,
  `.home`, and `.localdomain`
- AWS ARNs and EC2-style instance ids
- local paths and secret-like markers

The controlled error says the status source contains a non-public marker. It
does not echo the private endpoint or local path.

## Security and privacy boundary

- Network status fetch is disabled by default.
- Network status fetch, if used, must be explicit HTTPS and allowlisted.
- Public status JSON is a reduced user-facing feed, not a monitoring dump.
- Private runtime inventory remains excluded.
- Local file status-source errors do not print stack traces.
- `private_runtime_details_included` remains `false` only when the public
  payload has passed the non-public marker checks.
- Google auth remains dry-run/contract only.
- OpenAI shared traffic remains disabled by default.
- Local private memory and local node content are not uploaded by this release.

## Known limitations

- This is a prerelease status/API bridge.
- It is not full YonerAI production.
- The official AWS backend is not deployed from the public repo.
- status.yonerai.com production publishing is not proven by this release.
- Production Oracle/control-plane runtime is not included.
- Production Google login/passkey/account runtime is not included.
- Live Discord is not restored.
- OpenAI shared traffic is not enabled.
- Production cloud memory sync is not included.
- Production installer signing/trust store is not included.
- npm, winget, and auto-update apply are not included.
- `src/cogs/ora.py` remains a legacy boundary and is not solved by this release.
