## Summary

YonerAI v0.16.0-alpha.1 makes the public CLI easier to extend safely. The
large CLI entrypoint is split into command, screen, service, and TUI modules;
policy status is exposed as a first-class public surface; and first-launch
onboarding now separates language selection from Google auth dry-run guidance.

This prerelease also includes the current-main status security fix from PR
#509: explicit network status fetches no longer follow redirects, so a public
status URL cannot redirect the CLI toward loopback or private endpoints.

## What users can run

| Command | What it does | What it does not do | Mode |
| --- | --- | --- | --- |
| `yonerai` | Opens the interactive CLI home screen with the modularized command runtime. | Does not call production cloud, Oracle, Discord, or live providers by default. | Local interactive |
| `yonerai chat --lang ja` | Opens the Japanese-first chat entry point. | Does not enable production Google login or upload local private memory. | Local interactive |

## Runnable changes

- Split the public CLI into focused modules under `clients/cli/yonerai_cli/commands`,
  `screens`, `services`, and `tui`.
- Kept `clients/cli/yonerai_cli/cli.py` as a thin entrypoint instead of a
  long-term feature bottleneck.
- Added policy runtime objects for provider, model, pricing, permission,
  runtime, update, memory sync, and cloud escape policy.
- Added `yonerai policy status --pretty/--json`.
- Added TUI `/ポリシー` and `/policy`.
- Added first-launch auth onboarding after language selection. It is dry-run
  guidance only and is recorded locally as seen after the user chooses.
- Added packaged fallback support for `yonerai demo` and `yonerai smoke` so
  package-style executions do not require repo-only script files.
- Hardened `yonerai api` and `yonerai policy` error handling so missing core
  fixtures return controlled CLI errors instead of raw import failures.
- Fixed `yonerai install status` so reports without an `ok` field do not crash.
- Added `docs/architecture/RUST_BOUNDARY_PROPOSAL.md` to define what may move to
  Rust later and what should stay Python for now.
- Included PR #509 redirect hardening for Status API status-source fetches.

## Policy runtime boundary

Policy is now visible as runtime state instead of scattered UI constants.

Current policy surfaces include:

- provider selection state
- model selection state
- pricing status
- permission and approval status
- runtime mode status
- update policy state
- memory sync policy state
- cloud escape policy state

The policy report is observational in this prerelease. It does not grant new
permissions, enable live provider traffic, enable production cloud, or upload
private content.

## Architecture boundary

The CLI split is behavior-preserving architecture work plus visible policy and
onboarding surfaces. It does not solve every long-term boundary.

Confirmed:

- `cli.py` is reduced to a thin parser/dispatcher entrypoint.
- command modules own command-specific parser and handler logic.
- services own local config, package fallback, memory, and update support.
- screens/TUI modules own interactive rendering and palette behavior.

Still intentionally not claimed:

- `src/cogs/ora.py` remains a legacy boundary.
- broad ORA namespace migration is not complete.
- uncontrolled autonomous agents are not added.
- production Oracle/cloud runtime is not added.

## Status redirect hardening

Explicit network status fetch remains opt-in. When it is used, YonerAI now
rejects redirects instead of following them. This prevents an allowlisted public
status source from redirecting the CLI to loopback, private IP ranges, metadata
service endpoints, or other private runtime details.

## Known limitations

- This is an alpha prerelease line, not the stable CLI Local Runtime line.
- Production Oracle/cloud runtime is not included.
- Production AWS backend is not included.
- Production Google login/passkey/account runtime is not included.
- Live Discord is not restored.
- OpenAI shared traffic is not enabled.
- Local private memory is not uploaded automatically.
- Production cloud memory sync is not included.
- Production installer signing/trust store is not included.
- npm, winget, and auto-update apply are not included.
- Arbitrary shell/file/tool execution is not enabled.
- `src/cogs/ora.py` remains a legacy boundary and is not solved by this release.
- `reference_clawdbot` is not touched.
