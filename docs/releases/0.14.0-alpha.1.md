## Summary

YonerAI v0.14.0-alpha.1 defines the public contract for a future official
external API used by yonerai.com and private/AWS backend work. The public
repository now exposes schemas, fixture-only status, rate-limit policy, sync
privacy boundaries, and AWS handoff documentation without adding production
cloud runtime or secrets.

This release also includes the current-main live-boundary fix from PR #497 so
read-only and deny safety profiles display the effective live-provider state
after local policy suppression.

## Official API boundary

The public repository now defines the official API shape. It does not implement
the production backend.

Public repo state:

- contract and schema files
- fixture-only CLI/TUI status
- conformance tests
- sync privacy policy
- rate-limit policy
- private/AWS handoff document

Private or official lane required later:

- real AWS/API Gateway/CloudFront/WAF deployment
- real account backend
- real production Google login
- production Oracle worker
- production rate-limit storage and enforcement
- production secrets management
- production observability and incident response

## Auth and sync boundary

- Google login remains dry-run/contract only in the public repo.
- PKCE, state, loopback redirect, minimal `openid email profile` scopes, and
  no plaintext refresh-token storage are the contract boundary.
- Cloud-to-local sync remains contract/preview only unless a future official
  account flow is linked.
- Local-to-cloud sync remains disabled by default.
- Local-to-cloud requires explicit approval and audit reason.
- `local_private`, `local_only`, `secret_like`, private file content, local node
  payloads, local memory content, provider keys, and local absolute paths are
  excluded from public cloud fixtures.

## Rate-limit boundary

The public contract defines the headers and quota buckets that official backend
implementations must honor:

- `Retry-After`
- `X-YonerAI-RateLimit-Limit`
- `X-YonerAI-RateLimit-Remaining`
- `X-YonerAI-RateLimit-Reset`
- `X-YonerAI-RateLimit-Bucket`

Quota exceeded responses use HTTP `429`, a `quota_exceeded` code, and a
required `retry_after` field. When safe, the CLI may fall back to local mock or
loopback local LLM behavior. It must not upload private content or enable shared
traffic to bypass quota.

## Known limitations

- This is a prerelease contract slice.
- It is not full YonerAI production.
- The official API is not deployed from the public repo.
- The AWS backend is a handoff, not deployed infrastructure.
- Production Oracle/control-plane runtime is not included.
- Official Managed Cloud runtime is not included in this public repository.
- Production Google login, passkey, and account linking are not enabled.
- Live Discord gateway restoration is not included.
- OpenAI shared traffic is not enabled.
- Local private memory sync up is not automatic.
- No production signing key or production trust store is included.
- No production network installer, npm channel, or winget channel is included.
- SafeShell remains plan/diagnostic bounded behavior, not arbitrary shell.
- Workspace File Access Guard is not arbitrary file access, folder crawling,
  PDF/image parsing, or a full file-summary product.
- Self-evolution remains proposal-only and synthetic/low-resolution in public.
- `src/cogs/ora.py` is not solved by this release.
- `reference_clawdbot` is not touched.
