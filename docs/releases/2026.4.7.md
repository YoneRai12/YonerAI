## Highlights (JP)
- **公開ノードの初回セットアップを復旧**:
  - `requirements.txt` から `chromadb` を外し、Windows の clean 環境で base install が止まりにくい形へ変更
  - Microsoft Visual C++ Build Tools がなくても、公開ノード本体の導入を進めやすくした
- **Semantic Memory を optional 化**:
  - `requirements-optional-memory.txt` を追加
  - ChromaDB を使う `VectorMemory` は必要な人だけ明示的に入れる方式へ変更
- **起動時の劣化動作を明確化**:
  - `chromadb` 未導入でも YonerAI 本体は起動継続
  - 使えないのは ChromaDB ベースの semantic recall のみ
  - エラーメッセージから optional requirements へ直接誘導
- **配布ドキュメントと回帰テストを追加**:
  - `README.md` と `docs/ENV_FILES.md` に optional memory 導線を追記
  - `VectorMemory` が optional dependency なしでも import-safe であることをテストで固定
  - `core-test` workflow の trigger 範囲を見直し、`main` 向け PR で required check が欠けないように修正

## Highlights (EN)
- **Restored clean-install bootability for the public node**:
  - Removed `chromadb` from the default `requirements.txt` install path
  - Base setup on Windows no longer depends on ChromaDB native build tooling
- **Made semantic memory explicitly optional**:
  - Added `requirements-optional-memory.txt`
  - ChromaDB-backed `VectorMemory` is now opt-in instead of blocking first boot
- **Clarified degraded-startup behavior**:
  - YonerAI still starts when `chromadb` is absent
  - Only semantic recall stays disabled
  - The runtime error points directly to the optional requirements file
- **Added release-facing docs and regression coverage**:
  - Updated `README.md` and `docs/ENV_FILES.md`
  - Added a regression test to keep `VectorMemory` import-safe without the optional dependency
  - Fixed the `core-test` workflow trigger scope so the required status check is present on every PR to `main`
