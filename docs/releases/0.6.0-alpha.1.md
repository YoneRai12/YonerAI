# 2026.05.27 — YonerAI v0.6.0-alpha.1 CLI TUI Runtime

## Summary

This alpha turns the local CLI runtime into a more usable Japanese-first terminal
experience. It adds slash command completion, Rich terminal panels/status where
available, model/provider/settings flows, local update checking, and a dry-run
PowerShell installer skeleton. It is still a CLI Local Runtime alpha, not full
YonerAI cloud production and not a production installer.

## What You Can Actually Run Now

| Command | What it does | What it does not do | Mode |
| --- | --- | --- | --- |
| `yonerai` | Starts the interactive CLI when stdin is a terminal. | Does not enable live providers, production cloud, live Discord, or arbitrary shell execution. | Local interactive |
| `yonerai chat` | Starts the same interactive screen explicitly. | Does not bypass safety settings or provider opt-in. | Local interactive |
| `/` | Shows slash command candidates inside the interactive shell. Japanese mode lists Japanese commands first. | Does not send `/` to a provider. | Local UI |
| `/設定` | Opens the settings summary. | Does not store provider keys or print local absolute paths. | Local UI |
| `/モデル` | Shows model preference and local LLM setup guidance. | Does not install models, probe non-loopback URLs, or send a prompt. | Local UI |
| `/提供元` | Shows provider readiness for mock, local LLM, OpenAI-compatible, Anthropic, and Gemini. | Does not call live providers. | Local UI |
| `/安全` | Shows network, file, shell/tool, live-provider, and cloud-escape boundaries. | Does not grant broad shell/tool/file access. | Local UI |
| `/履歴` | Shows redacted local run history when a ledger is configured. | Does not upload run history. | Local-only |
| `/タスク` | Shows current/recent classify/route/provider/execution/review/result progress. | Does not start uncontrolled subagents. | Local UI |
| `/エージェント` | Shows the deterministic reviewer/subagent plan. | Does not run autonomous subagents. | Plan display |
| `/更新` | Runs the local manifest update check from inside the TUI. | Does not download, install, mutate PATH, execute remote code, or require admin. | Dry-run |

## CLI TUI

- `prompt_toolkit` is now a normal CLI dependency, so compatible terminals get
  slash completion and Tab/arrow-key selection.
- Rich is now a normal CLI dependency, so compatible terminals get colored
  panels and a status spinner for interactive ask execution.
- Plain fallback remains mandatory: non-TTY, CI, pipes, unsupported terminals,
  or `--color never` keep readable line-oriented output.
- JSON output continues to avoid ANSI/control sequences.

## Japanese-First Commands

Japanese mode shows Japanese labels first:

```text
/設定
/モデル
/提供元
/安全
/履歴
/タスク
/エージェント
/更新
/終了
```

English aliases such as `/settings`, `/models`, `/providers`, `/safety`,
`/runs`, `/tasks`, `/agents`, `/update`, and `/quit` still work.

## Provider And Local LLM

- Mock provider remains ready by default.
- Local LLM guidance remains loopback-only: `localhost`, `127.0.0.1`, or `::1`.
- OpenAI-compatible, Anthropic, and Gemini remain opt-in readiness surfaces unless
  explicitly configured with live/provider-specific opt-in.
- Provider keys are not printed, stored in CLI config, or written to run history.

## Update And Installer Safety

- `yonerai update check` is a new quick update status command backed by local
  manifest validation.
- `yonerai update plan` remains the fuller dry-run update planner.
- `install.ps1` is a dry-run-only future installer skeleton. It exists so the
  public repository has a safe one-command installer entrypoint to harden before
  any network installer is considered.
- Production signing keys, production trust stores, npm, winget, and remote
  execute install flows are not included.

## Security And Boundary

- No production Oracle/cloud runtime.
- No live Discord or Discord token use.
- No deploy/public tunnel.
- No arbitrary shell/file/tool execution.
- No provider live call by default.
- Local LLM remains loopback-only.
- Official Managed Cloud remains external/contract-only.
- `src/cogs/ora.py` is not solved by this release.
- `reference_clawdbot` is not touched.

## Not Included

- Stable production-cloud YonerAI.
- Production Oracle/control-plane runtime.
- Official Managed Cloud runtime in this public repository.
- Live Discord gateway restoration.
- Production installer, remote execute installer, npm package, or winget channel.
- Production signing keys or production trust stores.
- Arbitrary shell, arbitrary local file, or uncontrolled tool execution.
- Default live external provider calls.
- Autonomous subagent execution.

## Known Limitations

- This is an alpha TUI runtime, not a stable production cloud release.
- Arrow-key completion depends on terminal support and `prompt_toolkit`; plain
  fallback remains available.
- Local LLM setup is guidance/status-first. Users must run their own loopback
  model server.
- Update check is local-manifest based by default, not a live remote updater.
- The PowerShell one-command installer is a dry-run skeleton, not a production
  installer.
- npm and winget channels are not ready.
- Production signing/trust material is not present in the public repository.
