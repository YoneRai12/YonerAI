# YonerAI v2026.5.20 Web UI Mock-chat Security Checkpoint

Date: 2026-05-20  
Status: public Web UI mock-chat checkpoint  
Scope: dependency-security triage / public Core message API hardening / `clients/web` mock-chat surface  
Not a production release

## Open

YonerAI public development has moved from a Core API-only message checkpoint to the first user-visible mock-chat surface.

This checkpoint is still credential-free and local. It is not live provider generation, not Google login, not persistent memory, not Discord gateway completion, and not production readiness.

## Closed

- Hardened `POST /v1/public/messages` follow-up issues:
  - route handler is synchronous because it does no async work
  - manual 400 responses now use a flat `{ "error": "...", "message": "..." }` schema
  - secret-like marker detection catches prefix/suffix variants such as `user_api_key`
- Triaged current Dependabot alerts into public-safe lanes.
- Updated the public Core dependency requirement for Pillow to address the critical Core alert.
- Updated `clients/web` Next.js to `16.2.6`.
- Removed unused `discord.js` from `clients/web`, removing the production `undici` high-alert path from the chosen Web UI surface.
- Added a `clients/web` mock-chat UI that posts to the existing public Core message endpoint through the local rewrite.

## Changed

- `clients/web` root page now renders a public mock/offline chat surface instead of the older run/SSE-oriented placeholder.
- `clients/web` layout no longer shows fake history, personal labels, or live-provider model names.
- README, README_JP, and the capability matrix now state that the current public MVP includes a Web UI mock-chat surface, while live AI chat remains out of scope.
- A public Dependabot triage report records fixed, deferred, and blocked dependency lanes.

## Security and hygiene

- The Web UI calls only the local public mock endpoint path.
- No OpenAI, Gemini, Anthropic, Discord, SNS, web search, or external provider call is added.
- No Google login, cookie-based identity flow, or memory persistence is added for the mock-chat surface.
- No private repository, private runtime inventory, live route map, control-plane detail, or operational ledger is included.
- This checkpoint does not rewrite git history and does not create a GitHub Release tag.
- Remaining `ora-ui` critical/high alerts are tracked as a separate dependency-security lane.
