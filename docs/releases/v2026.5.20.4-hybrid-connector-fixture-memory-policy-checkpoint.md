# v2026.5.20.4 Hybrid Connector Fixture and Memory Policy Checkpoint

Status: public checkpoint note. This is not a production release.

## Open

- Production signing and key lifecycle are still open.
- Durable replay protection is still open.
- Real private connector implementation is still open.
- Persistent memory remains unimplemented.
- Owner approval UI and official cloud auth remain open.

## Closed

- Added a synthetic Hybrid Connector Fixture contract.
- Added public fixture helpers that create memory candidate, self-evolution signal, and improvement proposal envelopes.
- Added tests proving valid signatures still route through policy and quarantine.
- Added memory candidate policy scaffold with quarantine-first behavior.
- Added capability priority map for upcoming design lanes.

## Changed

- Public hybrid helpers now support envelope mapping round-trip for fixture serialization.
- Donation policy now has explicit memory candidate policy checks.
- Improvement proposal donations require test, rollback, privacy, hype-debt, provider-independence, and same-experience evidence fields.

## Security and hygiene

- Signed payloads are not trusted by default.
- Memory candidates default to `memory_persisted=false`.
- Raw chain-of-thought, raw prompts/completions, private runtime inventory, live route maps, and secret-like payloads are rejected.
- Replay, wrong audience, expiry, hash mismatch, and invalid signature cases remain covered.

## Not included

- No production cloud deployment.
- No production DB.
- No real private data ingestion.
- No persistent memory.
- No Google login.
- No automatic self-evolution mutation.
- No full hybrid connector.

## Still open

- Private/local connector implementation with real owner-controlled signing.
- Production-grade key registry and revocation.
- Durable nonce store.
- Approval workflow UI.
- Capability/extension boundary hardening.

## Next

Recommended next lane: capability / extension boundary hardening before tools/MCP, swarm, identity, Discord, or final Web UI work.
