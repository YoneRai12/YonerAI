# 2026.06.02 - YonerAI v0.13.0-alpha.2 Agent Console Dogfood Patch

Release date: 2026-06-02
Tag: `v0.13.0-alpha.2`
Status: prerelease dogfood patch for the v0.13 Agent Console Runtime
Previous release: `v0.13.0-alpha.1`

## Summary

YonerAI v0.13.0-alpha.2 is a narrow dogfood stabilization patch for the Agent Console Runtime. It keeps the v0.13 interactive CLI surface, then fixes the permission/profile and preview mismatches found while dogfooding v0.13.0-alpha.1.

This is not a stable promotion. It does not enable production Oracle/cloud runtime, production Google login, live Discord, OpenAI shared traffic, arbitrary shell/tool execution, provider-key storage, automatic local private memory upload, production signing/trust, npm/winget, or a production network installer.

## What changed since v0.13.0-alpha.1

- `/権限 read-only` now clears existing live-provider and network permissions instead of preserving them from a prior `/live on` or `/network on`.
- `/権限 dry-run-only` now also clears existing live-provider and network permissions.
- Permission profile updates now batch config writes through one load/save path instead of several sequential disk writes.
- `/計画 <task>` now displays a planner preview for the task instead of only switching mode.
- `/レビュー <text>` now displays a reviewer preview for the text instead of only switching mode.
- `@researcher` preview coverage was added to the regression tests.

## Dogfood result

Verified dogfood paths:

- command palette: `/コマンド`, `/パレット`
- mode/profile: `/モード`, `/計画 <task>`, `/レビュー <text>`, `/権限 read-only`, `/権限 dry-run-only`
- subagent preview: `@planner`, `@reviewer`, `@researcher`
- memory: `yonerai memory status/add/list/forget/sync preview`, `/記憶`, `/メモリ`
- update: `yonerai update check`, `yonerai update plan`, `/更新`
- history/ledger: local redacted run ids and memory ids only

The dogfood result is good enough for an alpha patch. It is not enough for stable v0.7.0 promotion yet because the memory/agent console surface still needs more real user testing after this patch release.

## Safety and boundaries

- live providers remain explicit opt-in and are off by default.
- read-only and dry-run-only profiles now clear prior live/network permissions.
- public-safe subagent commands are previews only; they do not spawn real autonomous workers.
- local memory remains local by default.
- local-to-cloud memory sync remains disabled unless a future explicit approval lane enables it.
- run ledger output records run ids, route/provider decisions, and memory ids only.
- arbitrary shell, arbitrary file access, live Discord, production Google login, production Oracle/cloud runtime, production signing/trust, and OpenAI shared traffic remain out of scope.

## Known limitations

- This is a prerelease.
- It is not full YonerAI production.
- It is not a stable v0.7.0 promotion.
- Production cloud memory is not implemented.
- Production Oracle/control-plane runtime is not included.
- Official Managed Cloud runtime is not included in this public repository.
- Production Google login/passkey/account system is not enabled.
- Live Discord gateway restoration is not included.
- OpenAI shared traffic is not enabled.
- npm and winget channels are not available.
- Local private memory sync up is not automatic.
- SafeShell is not arbitrary shell execution.
- Workspace File Access Guard is not folder crawling, PDF/image parsing, arbitrary file access, or full file summarization.
- Persistent memory remains an alpha boundary runtime, not a complete product memory system.
- Self-evolution does not mutate code, GitHub, releases, deployments, or production configuration.
- `src/cogs/ora.py` is not solved by this release.
- `reference_clawdbot` is not touched.
