# YonerAI Public FAQ 2026-05-20

Status: public-safe FAQ for the v7.7 Growth/SNS lane.

## Is YonerAI production-ready?

No. The current public repo is a checkpointed local/public MVP surface with
contracts, tests, and smoke paths. It is not production-ready and not a deployed
official cloud product.

## What can I try today?

You can try the local Core API health check, mock/offline public message flow,
loopback-only local LLM message mode, the in-memory Surface API run contract, the
local smoke CLI, and the temporary Web Chat MVP / smoke-demo surface.

## Does YonerAI call external providers by default?

No. The public MVP does not require provider API keys. Local LLM mode is
loopback-only and rejects arbitrary remote provider URLs by default.

## Is the Web UI final?

No. `clients/web` is a temporary Web Chat MVP / smoke-demo surface. It is useful
for checking the public Core API path, but it is not the final product UI.

## Is the CLI final?

No. `clients/cli` is a local smoke CLI for health, message, and run checks. It is
not the final CLI product and not the native Japanese CLI.

## Why is the native Japanese CLI separate?

Japanese command intent can be ambiguous. The native Japanese CLI lane needs its
own confirmation, explanation, dry-run, approval binding, and refusal wording
instead of inheriting terse English CLI behavior blindly.

## Does YonerAI have persistent memory now?

No. Public session metadata and run state are in-memory checkpoint surfaces.
Hybrid memory candidates are quarantined first and require approval before any
future persistence decision.

## Is official cloud available?

No. The public repo has official cloud control-plane planning contracts and
non-production skeleton work in the control-plane lane, but no deployed official
cloud product is claimed here.

## Does self-evolution edit code automatically?

No. Public self-evolution is proposal-only and approval-gated. Donated
self-evolution signals are not allowed to create branches, patch code, merge, or
deploy automatically.

## Is Discord complete?

No. Discord gateway completion is not part of the current public checkpoint.
`src/cogs/ora.py` remains unresolved boundary residue and is not treated as
solved.

## How should I describe YonerAI publicly?

Use: provider-independent AI execution foundation, public local MVP checkpoint,
contract-first API/CLI/Web boundaries, and approval-gated self-evolution.

Avoid: production-ready, official cloud complete, ChatGPT-equivalent, full hybrid
connector, persistent memory complete, final Web UI, Google login complete, or
provider ecosystem complete.
