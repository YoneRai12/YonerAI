# YonerAI Demo Plan 2026-05-20

Status: claim-guarded demo draft for the v7.7 public surface ladder.

## Goal

Show the current public YonerAI checkpoint without implying production readiness,
official cloud availability, persistent memory, final Web UI, Discord gateway
completion, or provider ecosystem completion.

## Demo Spine

1. Start from the public README and current checkpoint link.
2. Show the current capability matrix.
3. Start local Core API in a development environment.
4. Verify `GET /health`.
5. Send a mock/offline message to `POST /v1/public/messages`.
6. Send a mock run to `POST /api/v1/agent/run` and inspect `events_url` /
   `results_url`.
7. Run the local smoke CLI:
   - `yonerai health`
   - `yonerai message --mode mock "hello"`
   - `yonerai run --mode mock "hello"`
8. Open `clients/web` only as the temporary Web Chat MVP / smoke-demo surface.
9. Point to the native Japanese CLI contract as the next UX-specific command
   lane, not as an implemented command runner.
10. Point to hybrid signed-envelope and donation-policy docs as safety boundary
    work, not a full hybrid connector.

## Demo Boundaries

Keep these statements visible or verbal:

- Local smoke path only.
- No production deploy.
- No official cloud product.
- No persistent memory.
- No Google login.
- No Discord chat completion.
- No web search.
- No external provider live generation by default.
- No final Web product UI.

## Safe Visuals

Safe visuals:

- README first screen
- capability matrix
- local Core API health output
- mock/offline message response
- CLI help and successful local smoke command
- temporary `clients/web` smoke surface
- public contract docs
- release-note checkpoint index

Avoid visuals that expose:

- private runtime inventory
- live route maps
- hostnames
- local machine paths
- credentials
- raw chain-of-thought
- production dashboards
- control-plane internals

## Demo Failure Handling

If a demo command fails:

- explain the failing local prerequisite
- show the relevant doc/test
- do not claim the capability works live
- do not switch to private/prod surfaces to compensate

Point users to:
