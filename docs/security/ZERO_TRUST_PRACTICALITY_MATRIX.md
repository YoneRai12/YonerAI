# Zero-Trust Practicality Matrix

Status: public-safe practicality matrix. This document maps current guardrails to usable MVP workflows. It is not a zero-trust completion claim, production security certification, or official-cloud completion claim.

| control / boundary | risk reduced | user friction added | practical compromise | current repo status | next implementation gate | what must not be claimed yet |
|---|---|---|---|---|---|---|
| Loopback-only local LLM provider URLs | Reduces accidental remote provider calls, SSRF-like routing, and model prompt leakage to public hosts. | Users must run Ollama, LM Studio, llama.cpp, text-generation-webui, or LocalAI locally on loopback. | Local Core API remains usable with `mock` / `offline`; local mode works when a loopback model server is present. | Implemented and tested in public Core provider path. | Model listing and clearer local error UX. | Complete provider ecosystem security. |
| No external provider calls by default | Prevents hidden OpenAI, Anthropic, Gemini, web search, SNS, or Discord calls in the public MVP. | Users do not get cloud model answers unless they bring a local server. | Keep public MVP credential-free and local-first. | Documented and covered by local provider tests. | Explicit provider routing contract for future official/private lanes. | External provider support or production model routing. |
| No provider API keys required for public MVP | Reduces secret handling risk in public checkout. | Users cannot use hosted provider keys through this public lane. | Public trial path uses health, mock/offline, and loopback local LLM only. | README and capability matrix describe this boundary. | Separate private/official credential boundary. | BYOK or official hosted provider completion. |
| Signed envelope does not equal trust | Reduces forged-origin and payload-tampering risk without trusting a signature blindly. | Donated payloads need policy checks after signature verification. | Signature proves origin/integrity only; policy decides reject/quarantine/queue. | Public schema/policy tests and control-plane skeleton exist. | Production key lifecycle and durable trust registry. | Production hybrid connector or production signing. |
| Donation / memory quarantine before persistence | Reduces memory poisoning, secret persistence, and self-evolution poisoning. | Useful donated memory cannot become durable memory immediately. | Memory candidates are quarantined first; approval/policy review remains required. | Fixture and policy docs/tests cover quarantine behavior. | Approval workflow and durable memory decision. | Persistent memory completion. |
| No persistent memory by default | Reduces private data retention and cross-session leakage. | Public sessions are temporary and in-memory only. | Conversation Session Scaffold returns `memory_persisted: false` while still grouping turns. | Public Core session scaffold is implemented. | Memory policy scaffold and owner-approved persistence model. | Cross-device history or long-term natural memory. |
| Approval-gated self-evolution | Reduces unapproved code mutation, branch creation, deploy, and product drift. | Improvements require review rather than automatic application. | Public self-evolution remains proposal-only; official queue is synthetic/non-production. | Public proposal-only path and control-plane skeleton are documented. | Approval UX and bounded implementation lane. | Automatic self-evolution mutation. |
| Raw chain-of-thought exclusion | Reduces leakage of private reasoning, prompts, and sensitive internals. | Public trace UX must use summaries/labels instead of raw internals. | Show safe reasoning summaries and already-public sources only. | Public docs and trace guardrails state the boundary. | Trace contract enforcement in all future surfaces. | Complete observability or raw reasoning access. |
| Public/private/control-plane separation | Reduces private runtime and operational exactness leakage. | Some implementation details stay out of public docs. | Public repo publishes contracts and fixtures, not private/control-plane internals. | Cross-repo contract docs exist. | Dedicated repo-boundary validation and private/control-plane review. | Repo split completion or operational exactness. |
| No production signing keys in public repo | Reduces key leakage and trust-store compromise. | Public tests use fixtures/protocols only. | Use test-only deterministic fixture behavior where needed. | Public hybrid tests avoid production keys. | Control-plane/private key lifecycle decision. | Production trust store readiness. |
| No production trust store in public repo | Reduces accidental public trust of operational nodes. | Trusted issuer state is not available in public checkout. | In-memory/test registries support policy tests. | Control-plane skeleton is non-production. | Durable trust registry design. | Official node trust completion. |
| No Google login yet | Reduces premature identity and OAuth risk. | No cross-device account identity in public MVP. | Session scaffold remains local/non-authenticated. | README non-claims cover login. | Identity contract and OAuth security review. | Google login completion. |
| No deploy claim | Reduces operator confusion and accidental production assumptions. | Users must run locally for public MVP. | Docs show local smoke paths only. | Release notes and README avoid deploy claims. | Separate deployment lane with owner approval. | Official cloud or live-ops completion. |
| Web client smoke surface | Reduces final-UI overclaim while preserving a usable demo. | UI remains temporary and limited. | `clients/web` can call safe Core API modes without becoming final product UI. | Temporary Web Chat MVP is documented. | Final Web UX/product lane. | Final Web UI completion. |
| Discord not completed | Reduces mismatch between legacy Discord code and current public MVP claims. | Discord users do not get a completed public gateway claim. | Discord remains a separate lane; public Core grows independently. | README keeps Discord out of current MVP claims. | `src/cogs/ora.py` extraction and gateway contract. | Discord gateway completion. |
